# ADR 025: 공통(단체) 판정 머릿수에 실인물 자격 게이트 — 오검출 얼굴 제외

- 상태: 채택 (2026-07-20)
- 관련: CHMO-330(단체 사진 공용 노출), ADR 013/015(검출 오검출 필터), ADR 022(주 인물 비율 규칙)

## 문제

event 93(group 36)에서 혼자 나온 셀피(퍼 후드 파카)가 본인 인물 앨범 외에 **공용 앨범에도 노출**됐다.
원인은 YuNet이 프레임 왼쪽 끝의 퍼 후드 털 뭉치를 얼굴로 오검출(678px, score 0.652)한 것 —
CHMO-330의 단체 판정("주 인물 얼굴 2명+ → 공용")이 이 오검출을 주 인물로 세어
(678 ≥ 본인 얼굴 1088의 50%) 1인 사진을 단체 사진으로 오판했다.

이 오검출은 기존 검출 필터를 전부 통과한다:

| 필터 | 값 | 판정 |
|---|---|---|
| score 게이트 (0.6) | 0.652 | 통과 |
| 배경 크기 필터 rel_w 2.5% (ADR 013) | 16.8% | 통과 |
| 결합 필터 score<0.78 AND 종횡비<0.70 (ADR 015) | 종횡비 0.96 | 통과 |

화장품 팔레트 그림(다음 구현 목표 0.5)과 같은 "크고 그럴듯한 오검출" 부류로, 검출 단계 신호로는
못 거른다.

## 판별 신호 — 오검출의 임베딩은 모두와 바닥 유사도

오검출(털·사물)의 임베딩은 쓰레기라 event 내 **어떤 얼굴과도 닮지 않는다**. 실측(S3 이벤트 13개,
얼굴 309개, 미배정 121개)의 "event 내 최근접 유사도(max-sim)":

- 퍼 후드 오검출: **0.183** (앨범 centroid와는 0.104)
- 주 인물 크기 실인물 미배정 최저: **0.191** (event 90 고개 숙인 옆얼굴 — 육안 검증, 진짜 2인 사진)
- 그 위 실인물: 0.239(event 93 고개 숙인 2인), 0.265, 0.284 … 연속 분포
- 클러스터 배정 얼굴 최저: 0.407

빈 구간 (0.183, 0.191)에서 **실인물 보호 쪽(하단)으로 0.185 채택**. 오검출을 놓치는 비용(1인 사진이
공용에 남음 = 현상 유지)보다 실인물을 제외하는 비용(진짜 2인 사진이 공용에서 빠짐 = 새 회귀)이
크므로 바닥에 붙인다.

## 결정

`_assemble_result`의 주 인물 머릿수 카운트에 실인물 자격을 추가한다
(`ClusterConfig.common_face_min_similarity`, `CLUSTER_COMMON_FACE_MIN_SIMILARITY`, 0=비활성):

- **미배정(노이즈·저신뢰) 얼굴이 event 내 어떤 얼굴과도 유사도 < 0.185**면 오검출로 보고
  머릿수(및 그 사진의 최대 얼굴 폭 계산)에서 제외한다.
- 클러스터 배정 얼굴은 항상 센다 (실측 최저 0.407로 안전 + 사람이 확인 가능한 대상).
- 비교 상대가 없는 단독 얼굴(event 전체 1개)은 판단 근거가 없으므로 센다.
- **라우팅 전용** — 군집·저장(.npz)에는 영향 없음 (common_main_face_ratio와 같은 계층).

검출 단계 제거가 아니라 라우팅 제외인 이유: 검출 단계에는 임베딩이 없고(신호가 임베딩에만 있다),
행이 남아 있어야 이후 사용자 보정(reassign)·재보정의 대상이 된다.

## 검증

- 자가검증: `python -m app.handlers` 44건 통과 — 신규 ⑱(인물+오검출 사진은 단체 아님 + 게이트
  0(비활성)이면 공용 노출 재현하는 적대적 대조), 기존 행인·단체·일행 시나리오 전부 유지.
- 실 이벤트 13개 회귀 (before/after 전체 결과 diff): **클러스터 멤버십 전 이벤트 불변**, 변화는
  공용 목록 3건뿐이고 전부 의도된 수정 —
  - event 93: 퍼 후드 셀피가 공용에서 빠짐 (인물 앨범 소속 유지). 진짜 2인 사진·단체 사진은 공용 유지.
  - event 84·87 (폭 미상 w=0 레거시 행): 프레임에 잘린 블러 턱 조각(0.160)·흔들린 소형 얼굴(0.176)이
    붙은 사진 3건이 공용에서 빠짐 — 같은 코퍼스의 폭 보유 이벤트(88·92)에서는 크기 게이트(ADR 022)가
    이미 제외하던 얼굴들이라 미러 이벤트와 동작이 정렬된 것.
  - event 90·91: 무변화 — 실인물 0.191(고개 숙인 옆얼굴)의 2인 사진이 공용에 유지 (0.2였다면 회귀).

## 한계

- **보정 표본이 오검출 1건(퍼 후드)**: 빈 구간 (0.183, 0.191)의 폭이 0.008로 얇다. 실인물 쪽(0.191,
  고개 숙임·가림 얼굴)이 더 내려오는 리포트가 쌓이면 이 게이트로는 못 가르므로, 그때는 max-sim 단일
  축이 아니라 검출 score 병행(.npz에 score 저장 필요) 같은 결합 신호로 재설계한다.
- 오검출이 event 내 다른 오검출과 닮는 경우(같은 사물이 반복 등장) max-sim이 올라 게이트를 통과할 수
  있다 — 현 코퍼스에는 없음.
- 팔레트 그림 오검출(목표 0.5)의 앨범 형성 문제 자체는 별개로 남는다 — 이 게이트는 공용/단체 판정만
  고친다.
