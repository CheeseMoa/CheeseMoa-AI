# ADR 028: 크기 인지형 confident score 게이트 — 대형 오검출의 유령 앨범 해소

- 상태: 채택 (2026-07-22)
- 관련: CHMO-403, ADR 017(대형 저score 재검출 회복 — 이 결정의 전제), ADR 013/015(검출 필터),
  ADR 025(머릿수 실인물 자격), ADR 027(파편 이중 검출 디둡), 다음 구현 목표 0.5(얼굴형 그림/조형물 오검출)

## 문제

event 115에서 사람은 2명인데 인물 앨범이 **3개** 생겼다. 3번째 앨범은 다른 사람이 아니라 **손 브이(V)
포즈를 YuNet이 얼굴로 오검출**한 것이고, 그 앨범 썸네일이 손으로 뜬다. 손 오검출이 근중복(재업로드)된
두 사진에서 반복 검출돼(서로 유사도 0.590) min_cluster_size=2를 채우고 별도 인물 앨범으로 승격됐다.

원인은 검출 게이트다. YuNet(WIDER FACE 학습)은 손의 그늘 패턴을 얼굴로 **약하게** 오반응해
score 0.640/0.656을 줬는데, 이 값이 통과 게이트 0.6을 **턱걸이로 넘었다**. 기존 방어막이 전부 미발동:

| 방어막 | 손 오검출 값 | 결과 |
|---|---|---|
| ADR 013 크기 필터 (rel_w ≥ 2.5%) | rel_w 19.6~22.1% | 통과 (큼) |
| ADR 015 결합 필터 (score<0.78 AND 종횡비<0.70) | 종횡비 0.78·0.87 | 미발동 (종횡비 ≥0.70) |
| ADR 017 회복 (저score<0.6 대형 재판정) | score 0.64 ≥ 0.6 | 대상 아님 (confident로 직행) |
| ADR 025 머릿수 자격 (최근접<0.185 제외) | 근중복 쌍 0.590 | 통과 (서로 닮음) |
| ADR 027 파편 디둡 (같은 얼굴 파편) | 두 사진의 별개 검출 | 대상 아님 |

## 왜 이제 게이트를 올릴 수 있나 (ADR 017이 연 문)

과거에 게이트 상향이 막혔던 이유는 **YuNet이 초근접 대형 실얼굴에도 저score를 준다**는 것이었다 —
게이트를 올리면 진짜 얼굴이 먼저 떨어진다. **ADR 017이 이 문제를 풀었다**: 저score 대형 얼굴을 정규
스케일로 재검출하면 실얼굴은 score가 오르고(≥0.86) 오검출은 재검출이 실패한다. 즉 대형 얼굴에 한해
**"원본 스케일 score"와 "keep/drop 판정"이 분리**됐다. 따라서 대형 얼굴은 게이트를 올려도 회복이
백스톱이라 실얼굴을 안 잃는다. **단 이 백스톱은 대형(rel_w ≥ 0.20)에만 있다** — 소형은 회복 대상이
아니라 게이트 상향이 그대로 손실이 된다.

## 실측 (전 코퍼스 1,181장, `scripts/sweep_score_gate.py`, 2026-07-22)

게이트 0.6 통과 얼굴 전체를 뽑고 후보 게이트를 시뮬레이션(대형은 회복 재판정):

- **대형(rel_w ≥ 0.20)** [0.60,0.70): 실얼굴 **41개 전원 회복**(재검출 0.86~0.94, 초근접 아기·아동·
  성인 육안 확인), 오검출 **2개만 폐기**(event 115 손·event 5 손글씨 장부 — 재검출 −1.00 실패).
  → **실얼굴 손실 0, FP만 제거.**
- **소형(rel_w < 0.20)** [0.60,0.70): 흐린 아기·아동 실얼굴 ~7-8개(ev117 유아, ev81 자는 소년 등)가
  오검출(ev27 애니 그림 0.686 등)과 **score 구간이 겹친다**(실얼굴 최저 0.605 vs 그림 0.686 — 빈 구간
  없음). 회복 백스톱도 없어 게이트 상향은 이 실얼굴들을 소리 없이 폐기한다. ornament 조사가 부딪힌
  "소형 오검출은 값싼 신호로 못 거름"과 같은 벽.
- **게이트 값 0.70**: 제거되는 대형 FP 수가 0.70·0.72·0.75에서 전부 2개로 동일 — 0.70 위로 올려도 FP는
  더 안 잡히고 실얼굴만 불필요하게 재검출을 더 거친다(회복유지 41→86). 0.70이 최소비용 최적점.

## 결정

confident 게이트를 **얼굴 크기에 따라 다르게** 건다:

- **대형(rel_w ≥ big_face_rel_width)**: `big_face_confident_score`(0.70) 이상이어야 confident.
  [score_threshold, 0.70) 저score 대형은 ADR 017 회복 경로로 재판정(재검출 ≥0.80이면 유지, 아니면 폐기).
- **소형(rel_w < big_face_rel_width)**: 기존 `score_threshold`(0.6) 유지.

`FaceDetector.__init__`이 `_large_gate = max(score_threshold, big_face_confident_score)`를 계산하되
**big_face 회복이 켜져 있을 때만** 적용한다(회복 OFF면 백스톱이 없어 상향은 위험 → 기본 게이트).
`detect()` 게이트 루프가 얼굴 크기로 분기한다. `big_face_confident_score == score_threshold`이면 구 동작.

검증(`scripts/verify_size_gate.py`, 구동작 0.6 vs 신규 0.70 두 FaceDetector 전 코퍼스 diff):
event 115에서 대형 손(0.640/22.1%) 폐기·진짜 두 얼굴 유지·추가 0. 전 코퍼스에서 소형 폐기 0·추가 0,
대형 폐기는 손·장부 오검출뿐(대형 실얼굴 손실 0).

`DETECT_BIG_FACE_CONFIDENT_SCORE`(0.6과 같게 두면 비활성=구 동작).

## 한계

- **예방적 수정**: 검출 단계라 앞으로의 classify에만 적용된다. event 115의 .npz에 이미 저장된 손 임베딩
  행은 남아, 그 앨범은 재검출(재업로드/재처리) 전까지 유지된다 — recluster는 저장 임베딩을 쓰므로
  재군집만으론 안 사라진다(ADR 027 잔여와 동류).
- **소형 대형오검출은 여전히 못 잡는다**: event 115의 다른 손(rel_w 19.6%, 소형)은 그대로 통과한다.
  다만 단독 얼굴이라 min_cluster_size=2 미달 → 인물 앨범 미형성(uncertain/노이즈로 격리)이므로 유령
  앨범은 사라진다. 소형 손·그림·조형물 판별은 검출·유사도와 다른 축(재질·문맥)을 보는 별도 모델
  영역으로 남긴다(다음 구현 목표 0.5, ornament 조사 결론).
- 표본: 대형 FP 양성이 2개(손·장부)로 얇다 — [0.70,1.0)에 대형 실얼굴이 몰려 있어(median 0.906) 상향
  여지는 크지만, 리포트 축적 시 대형 FP score 분포를 재확인해 값 재보정. 도구
  `scripts/sweep_score_gate.py`·`scripts/verify_size_gate.py`.
