# ADR 014: 얼굴 미검출 fallback의 방향성 블러 검출 — 그라디언트 방향 쏠림

## Status

Accepted (2026-07-16). 흔들림 품질 게이트(CHMO-172)의 fallback 경로 보강 — 얼굴 경로의 판정
(주 인물 얼굴 crop variance)은 건드리지 않는다.

## Context

흔들림 판정은 두 경로다. 얼굴이 검출되면 주 인물 얼굴 crop의 정규화 Laplacian variance로 판정하고
(test2 라벨셋 13/13 정답), 얼굴이 하나도 없거나 전부 극소 얼굴이면 전체 이미지 variance < 100
fallback으로 넘어간다. 심하게 흔들린 사진일수록 얼굴 검출 자체가 실패하므로, 정작 가장 흔들린
사진들이 fallback으로 흘러간다.

문제: 전체 이미지 Laplacian variance는 **선명도가 아니라 장면의 텍스처 양을 재는 지표**다.
흔들렸어도 배경에 무늬·글자·조명이 많으면 값이 높게 나온다. test2 라벨셋의 흔들린 사진 중
얼굴 미검출 2장이 variance 117, 414로 임계 100을 넘어 둘 다 통과했다(414짜리는 임계를 4배
올려도 못 잡는다 — 임계 조정으로 해결 불가).

### 대체 지표 실측 (2026-07-15, test2 라벨셋: 흔들림 6장 · 선명 9장)

전 지표를 긴 변 1024 축소 후 측정(해상도 정규화):

| 지표 | 흔들림 범위 | 선명 범위 | 분리? |
|---|---|---|---|
| 정규화 전체 variance | 7.8~55.5 | 20.5~236.9 | ✗ 겹침 |
| 64px 패치 variance 상위 5% | 16.8~385.3 | 72.5~863.0 | ✗ 겹침 |
| **그라디언트 방향 쏠림** | **0.397~0.631** | 0.054~0.312 (예외: child4 0.491) | **○** |
| 평균 에지 폭 | 9.5~11.3 | 8.2~10.4 | ✗ 겹침 |

방향 쏠림이 유일하게 분리된다. 원리: 손떨림은 카메라가 찰나에 한 방향으로 움직인 것이라 모든
에지가 같은 방향으로 번지고, 이미지 전체의 그라디언트 방향이 쏠린다. 텍스처 양과 직교하는
신호라 variance가 놓친 사진(117, 414)을 둘 다 0.4대로 잡았다. 유일한 예외 child4(선명인데 쏠림
0.491 — 단순 구도로 에지 방향이 우연히 쏠림)는 정규화 variance 236.9로 명백히 선명하다.

## Decision

`shake_signals()`(`app/pipeline/quality.py`)가 (정규화 variance, 방향 쏠림)을 재고, fallback에서
기존 규칙에 **OR로 추가**한다 (`deps.build_face_extractor`):

> blurry = (전체 variance < 100) **OR** (쏠림 ≥ **0.40** AND 정규화 variance < **60**)

- 측정: 그레이스케일 → 긴 변 1024 축소(INTER_AREA) → 쏠림은 Sobel 그라디언트 크기 상위 10%
  픽셀의 구조 텐서 평균에서 `sqrt((Jxx−Jyy)² + 4·Jxy²) / (Jxx+Jyy)` (0=등방, 1=한 방향).
- 쏠림 임계 0.40: 선명 최고 0.312와 마진 0.088. 라벨셋 중간값이 아닌 보수 선택 — 흔들림 최저
  0.397짜리 1장은 **알려진 미탐**으로 남긴다(15장 샘플의 얇은 마진에 과적합하지 않기 위함).
- variance 상한 60: child4류(쏠림 높지만 매우 선명) 오탐 가드. 흔들림 실측 최고는 55.5.
- 기존 variance<100 규칙 유지: 완전히 뭉개진 사진(variance 1~7)을 잡는 검증된 안전망. OR 추가라
  기존 검출이 줄어드는 회귀는 구조적으로 없다.
- 설정: `QUALITY_SHAKE_COHERENCE_THRESHOLD`(0=비활성) · `QUALITY_SHAKE_MAX_NORM_VARIANCE`.

결과: test2 fallback 경로 흔들림 검출 0/2 → **1/2**, 선명 오탐 0 유지 (라벨셋 전체 14/15).

## 기각한 대안

- **fallback 임계 상향** (100 → 120+): 117짜리는 잡지만 414짜리는 불가능하고, 민무늬 벽 등
  단순 선명 장면의 오탐이 늘어난다. 지표 자체가 틀린 문제라 임계로는 못 푼다.
- **패치별 선명도** ("어딘가 선명한 패치가 있으면 선명"): 흔들린 사진도 고대비 에지 패치의
  variance가 높게 나와(최대 385) 분리 실패.
- **평균 에지 폭**: 흔들림/선명 분포가 겹친다(9.5~11.3 vs 8.2~10.4). Canny 에지 기준 프로파일
  측정의 노이즈가 커서 신호가 약하다.
- **블러 분류 CNN**: 모델 추가는 소싱·라이선스·적재 비용이 드는데, 적용 범위가 얼굴 미검출
  사진의 공통 앨범 ↔ blurry 라우팅뿐이라 비용 대비 과하다. OpenCV 연산만으로 충분.

## 재보정: 쏠림 임계 0.40 → 0.35 (2026-07-17)

한계에 예고한 재보정을 실행했다. event 58(라벨셋 업로드 재현)에서 알려진 미탐 1장(쏠림 0.397)이
실서비스 공통 앨범으로 유출된 것이 계기. 그 사이 실 이벤트 조사(event 50·55·57·58, ADR 018)로
**얼굴 미검출 + 선명** 사진의 쏠림 실측이 쌓였다: 0.118, 0.263, 0.263, 0.312 — 전부 0.35 미만이고,
유일한 예외 child4(0.491)는 기존 variance 가드(norm_var 236.9 ≥ 60)가 차단한다. 0.35는 미탐
0.397을 마진 0.047로 잡고 선명 최고 0.312와 마진 0.038을 유지하며, ADR 018의 해제 바닥(0.35)과
값이 정렬된다("쏠림 0.35 이상 = 흔들림 방향성 있음"이 잡기/풀기 양방향에서 일관).
결과: test2 라벨셋 fallback 검출 1/2 → **2/2**(전체 15/15), 선명 오탐 0 유지.

## 한계 · 리스크

- **샘플 15장 소규모** — 임계 0.40/60은 재보정 전제의 초기값이다. 얼굴 미검출+흔들림 라벨이
  10~20장 쌓이면 스윕 재실행할 것 (`python -m app.pipeline.quality`가 fallback 사진의
  norm_var·coherence를 출력하므로 실서비스 사례를 그대로 라벨로 축적 가능).
- **알려진 미탐 1장**(쏠림 0.397): 임계 바로 아래. 재보정 때 0.35~0.40 구간을 다시 볼 것.
- **회전 손떨림**: 카메라가 직선이 아니라 회전하면 번짐 방향이 위치마다 달라 전역 쏠림이 낮게
  나올 수 있다 — 전역 구조 텐서 방식의 원리적 한계 (라벨셋에는 미출현).
- 부분 블러(앞사람만 모션블러)·장노출 빛궤적은 여전히 못 잡는다 — fallback 이전 얼굴 경로와
  공유하는 기존 한계.
- 오분류의 영향 범위는 얼굴 미검출 사진의 공통 앨범 ↔ blurry 분류뿐 — 인물 앨범은 무영향.
