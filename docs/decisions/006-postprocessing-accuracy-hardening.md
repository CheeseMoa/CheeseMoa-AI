# ADR 006: 재군집 후처리의 정확도 보강 설계 (코드리뷰 반영)

## Status

Accepted

## Context

CHMO-164 클러스터링 구현에 대해 다각도 코드리뷰(파인더 10앵글 → 후보별 검증 → 갭 스윕)를
수행했고, 15건의 발견 중 5건이 **실행 재현된 정확성 결함**이었다:

1. 저신뢰 축출의 절대 바닥 분기가 사용자 split로 생긴 소형 클러스터를 통째로 비우고 `cluster_id`까지 은퇴시킴
2. 파편 병합의 전이 체인(A~B, B~C)이 서로 타인인 A와 C(유사도 0.147)를 한 앨범으로 융합
3. 균질 blob(중복 사진 버스트)은 HDBSCAN이 클러스터 0개를 내고, 병합·구제가 무력해 인물 앨범이 아예 안 생김
4. 비정규(단위 노름 아님) 임베딩이 후처리의 모든 유사도 임계를 조용히 우회
5. 노이즈 구제가 cannot-link 경합 시 유사도가 아니라 인덱스 순으로 배정 (더 나은 매치가 밀림)

이 문서는 각 결함에 대해 **어떤 대안 중 무엇을 왜 선택했는지** 기록한다.
나머지 10건(잠재 함정·메모리·문서 정합)의 처리는 각 코드 주석과 [ADR 005](./005-hdbscan-standalone-port.md)의
편차 기록을 참고.

## Decisions

### 1. 축출 보호는 "제약 당사자 얼굴" 단위 (결함 #1)

- **대안**: (a) cannot-link로 연결된 클러스터 전체를 축출 면제, (b) split로 생긴 클러스터에 별도 플래그, (c) 제약 당사자 얼굴만 보호
- **선택: (c)** — must-link 컴포넌트에 더해 cannot-link 쌍의 당사자를 `protected`에 포함한다.
- **이유**: 사용자가 직접 지목한 얼굴만이 "사람의 결정"의 실체다. 클러스터 전체 면제(a)는 제약과
  무관한 저품질 멤버까지 축출을 피하게 만들고, 플래그(b)는 순수 함수에 상태를 끌어들인다.
  당사자 보호만으로 재현 시나리오(2명 split 클러스터 전멸)가 해소되고, 비당사자 멤버의 품질 관리도 유지된다.

### 2. 파편 병합은 완전 연결(complete linkage) (결함 #2)

- **대안**: (a) 쌍별 임계(기존) 유지 + 임계 상향, (b) 병합할 때마다 병합체 centroid 재계산 후 재검사, (c) 완전 연결 — 병합될 두 컴포넌트의 모든 구성 클러스터 쌍이 임계 이상
- **선택: (c)** — 병합 전 centroid 유사도 스냅샷 위에서 교차쌍 전부를 검사한다.
- **이유**: (a)는 임계를 아무리 올려도 다리(bridge) 파편을 통한 전이 융합을 원리적으로 막지 못한다.
  (b)는 병합 순서에 따라 centroid가 이동해 결과가 순서 의존적으로 흔들린다. 완전 연결은 스냅샷만으로
  결정적이고, "병합체의 모든 구성원이 서로 동일 인물 수준"이라는 의미가 병합의 목적(같은 인물의 파편 회수)과
  정확히 일치한다. 같은 인물의 파편들은 교차쌍도 전부 높으므로(합성 검증) 정상 병합은 저해되지 않는다.

### 3. 클러스터 0개 퇴화는 "균질 blob 승격"으로 (결함 #3)

- **대안**: (a) `allow_single_cluster=True`, (b) 파라미터(eps 등) 재조정, (c) 전원 노이즈 + 전 쌍별 유사도가 동일 인물 수준일 때만 단일 클러스터로 승격
- **선택: (c)** — `_promote_single_blob`, 제약 강제 전에 실행해 cannot-link가 승격된 라벨도 분리할 수 있게 한다.
- **이유**: (a)는 실험으로 기각 — 단일 클러스터 모드의 소속 판정이 `cluster_selection_epsilon=0.15`
  (유사도 0.85 이내)와 상호작용해 실제 동일 인물 분산(~0.7)이 전원 노이즈가 된다. (b)는 PoC 검증
  레시피를 흔들며 TBD #4(실데이터 튜닝)의 몫이다. 승격 조건을 **전 쌍별 최솟값**(완전 연결과 동일 철학)으로
  잡아 낯선 두 얼굴(유사도 ~0)이 승격되는 오탐을 차단했고, 검증된 HDBSCAN 레시피는 건드리지 않는다.

### 4. 비정규 임베딩은 자동 정규화가 아니라 거부 (결함 #4)

- **대안**: (a) 입구에서 자동 재정규화, (b) 후처리마다 정규화 나눗셈 추가, (c) 검증 후 ValueError 거부
- **선택: (c)** — 노름이 1±1e-3 밖이면 거부 (허용 오차는 저장소 왕복 부동소수 오차 흡수용).
- **이유**: 파이프라인 계약상 임베딩은 embed 단계가 L2 정규화해 내보낸다. 비정규 입력은 계약 위반
  (호출자 버그: 저장소 직렬화 손상, 정규화 생략 등)이므로 detect·embed의 기존 철학(무변환·조기 거부)대로
  시끄럽게 실패시키는 게 조용한 오분류보다 낫다. (a)(b)는 버그를 숨긴다.

### 5. 노이즈 구제는 전역 유사도 내림차순 (결함 #5)

- **대안**: (a) 얼굴 인덱스 순 처리(기존), (b) 경합 감지 시에만 재정렬, (c) (얼굴, 클러스터) 후보 전체를 유사도 내림차순으로 처리
- **선택: (c)** — 동률은 (얼굴 인덱스 → 클러스터 순)으로 고정해 결정성 유지.
- **이유**: (a)는 cannot-link 경합에서 "먼저 줄 선" 낮은 유사도가 자리를 선점하는 역전이 재현됐고,
  독스트링의 순서 무관 주장도 거짓이었다. (b)는 경합 감지 로직이 별도로 필요해 더 복잡하다.
  전역 내림차순은 "더 나은 매치가 항상 먼저"라는 한 문장으로 정당화되고, 부수 효과로 얼굴별
  GEMV 루프가 1회 GEMM으로 바뀌어 성능도 개선된다.

### 6. 신뢰도는 계산 지점에서 노출 (발견 #12)

- **대안**: (a) 워커가 필요할 때 재계산, (b) 클러스터별 평균 신뢰도만 노출, (c) 멤버별 leave-one-out 유사도를 `PersonCluster.membership_similarities`로 노출
- **선택: (c)** — 저신뢰 축출 판정과 동일한 정의(`_loo_similarities` 공용 헬퍼)를 사용한다.
- **이유**: 외부 재계산(a)은 LOO 보정·cannot-link 마진 제외가 빠져 사용자가 보는 축출 결정과
  어긋난 신뢰도를 만든다. 스펙(§6.2 uncertain reason, §7 매칭 임계 설정값, TBD #2·#3)이 요구하는
  per-face 값이므로 판정 지점에서 그대로 내보내는 것이 유일하게 일관적이다.

### 7. 이식본 안전 수정의 허용 기준 (발견 #6·#8·#13)

- **대안**: (a) PoC 라인 단위 무변경 절대 유지, (b) 결과 라벨 비트 동일이 검증되는 안전 수정만 허용
- **선택: (b)** — 재귀→반복, 표본 2개 미만 사전 검증, in-place 연산 2건. 편차 목록은 ADR 005에 기록.
- **이유**: "무변경"의 목적은 결과 동일성 보장이지 라인 보존 자체가 아니다. 순수 파이썬 재귀는
  Cython 원본에 없던 크래시(RecursionError)를 들여왔으므로 오히려 원본 동작에 가까워지는 수정이고,
  세 수정 모두 sklearn 대조(30개 데이터셋 라벨 완전 일치)로 재검증했다. 단 편차는 반드시 ADR에
  남겨 이식 대조(diff) 시 혼란이 없게 한다.

## Consequences

- 후처리 파이프라인이 확정된다: 균질 blob 승격 → must/cannot-link 강제 → 파편 병합(완전 연결) →
  노이즈 구제(전역 내림차순) → 저신뢰 분리(LOO, 제약 당사자 보호) → ID 재조정 → 대표벡터.
- 5건의 재현 결함은 각각 회귀 테스트로 고정됐다 (합성 데이터 53케이스).
- `ClusterConfig`는 개별 범위에 더해 교차 불변식(`rescue_similarity ≥ min_membership_similarity`,
  `cluster_selection_epsilon ≤ 2`)을 강제한다 — TBD #3·#4의 실데이터 튜닝은 이 제약 안에서 움직인다.
- `PersonCluster`에 `membership_similarities`가 추가되어 결과 스키마(classify-result) 설계 시
  uncertain reason·신뢰도 필드의 원천으로 쓴다.
- 완전 연결 병합·승격은 보수적이라, 분산이 큰 동일 인물 파편(교차쌍 일부가 0.7 미만)은 병합을
  놓칠 수 있다 — 오병합(타인 융합)보다 미병합이 낫다는 정확도 우선 원칙의 의도된 트레이드오프이며,
  임계는 TBD #4에서 실데이터로 재조정한다.
