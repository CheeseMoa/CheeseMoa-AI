# ADR 018: 흔들림 재확인 게이트 — variance 판정을 방향 쏠림 바닥으로 재확인

## Status

Accepted (2026-07-17). 흔들림 품질 게이트(CHMO-172)의 최종 게이트 추가 — 얼굴 경로·전체 이미지
fallback의 variance 기반 blurry 판정에 공통 적용. ADR 014의 방향 쏠림 신호를 반대 방향(해제)으로
재사용한다.

## Context

EC2 실서비스에서 event 50(그룹 27, 앨범 205)의 사진 8장이 전부 blurry로 오분류됐다. 로컬 재현
(59장)으로 확인한 결과 8장 모두 **옛날 인화 사진을 폰으로 재촬영하고 앱이 블러 배경 프레임을
씌운 PNG**였다: 7장은 얼굴 경로(정규화 variance 10.0~24.6 < 임계 25), 1장은 얼굴 미검출 fallback
(whole_var 98.9 < 임계 100)으로 탈락했다.

원인은 지표의 본질적 한계다. Laplacian variance는 **잔결(고주파 디테일)의 양**만 재므로,
"흔들려서 디테일이 뭉개진 사진"과 "원판 자체가 소프트해 디테일이 원래 없는 사진"(인화지 해상력
+ 재촬영 손실 + 앱 스무딩)을 구분하지 못한다. 임계 조정으로는 해결 불가 — event 50에서 통과한
옛날 사진들도 26.1, 34.5, 37.4…로 연속 분포라 빈 구간이 없고, 임계를 내리면 진짜 흔들림
(라벨셋 최대 22.3)을 놓친다.

### 판별 신호 실측 (2026-07-17, test2 라벨셋 + event 50 오탐 7장)

손떨림은 모든 에지가 한 방향으로 번지지만(ADR 014), 소프트 원판의 디테일 부재는 등방이다.
variance가 흔들림이라 한 얼굴들에서:

| 신호 | 흔들린 얼굴 (라벨셋) | 옛날 사진 오탐 얼굴 | 분리? |
|---|---|---|---|
| 얼굴 crop 방향 쏠림 | 0.132~0.349 | 0.044~0.454 | ✗ 겹침 |
| **전체 이미지 방향 쏠림** | **0.444~0.631** | **0.100~0.268** | **○ (빈 구간 0.268~0.444)** |

얼굴 crop 쏠림이 판별력이 없는 이유: 흔들린 얼굴 crop은 그라디언트 자체가 약해 구조 텐서가
crop 경계·노이즈에 지배된다. 전체 이미지 쏠림은 배경 에지까지 포함해 손떨림의 전역 방향성을
안정적으로 잡는다.

## Decision

variance 기반 blurry 판정(얼굴 경로 + fallback 1차 규칙)의 **최종 게이트**로 전체 이미지 방향
쏠림 바닥(`shake_coherence_floor`, 기본 0.35)을 추가한다 — `quality.shake_confirmed()`,
`deps.build_face_extractor`의 판정 말미에 적용:

```
blurry(variance 판정) AND coherence < floor  →  blurry 해제 (소프트 원판)
```

- **0.35 채택 근거**: 오탐 최고 0.268과 흔들림 최저(얼굴 미검출 fallback 라벨 0.397, 얼굴 경로
  0.444) 사이의 빈 구간에서 미탐 리스크가 적은 쪽(낮은 값)을 선택. ADR 014의 임계 0.40과 다른
  값인 이유: 그쪽은 "쏠림만으로 흔들림 확정"(높아야 안전), 이쪽은 "쏠림 부재로 흔들림 기각"
  (낮아야 안전)이라 보수 방향이 반대다.
- ADR 014의 2차 신호(쏠림 ≥ 0.40)로 잡힌 사진은 정의상 게이트(≥ 0.35)를 통과하므로 영향 없다.
- `QualityConfig.shake_coherence_floor` / `QUALITY_SHAKE_COHERENCE_FLOOR`, 0 = 비활성(기존 동작).

## 검증 (2026-07-17)

- **event 50 (59장)**: 오탐 8장 전부 해제(육안 검수 — 전부 비흔들림 옛날 사진), 나머지 51장 판정 무변경.
- **test2 라벨셋 회귀 없음**: 흔들림 6장 중 5장 유지(나머지 1장은 ADR 014에 문서화된 기존 미탐,
  쏠림 0.397 — 이 변경과 무관), 선명 9장 전부 통과 유지.
- `python -m app.worker --smoke` 8건 전부 통과.

## 보강: variance 붕괴 면제 (2026-07-17, 당일)

배포 직후 event 55에서 **고스팅형 손떨림**(야간 셀피, 겹침 번짐) 사진이 게이트에 오해제되어 공통
사진첩으로 들어갔다: 얼굴 미검출 → whole_var 13.5 < 100으로 흔들림 후보였으나, 겹침 번짐은 에지가
한 방향이 아니라 쏠림이 0.306 < 0.35로 나와 게이트가 풀어버렸다. 쏠림은 **선형** 손떨림의 신호라
고스팅·회전 흔들림에는 낮게 나온다 — 게이트 단독으로는 이 부류를 전부 놓친다.

판별축은 variance의 **붕괴 수준**: 심하게 흔들린 사진은 잔결이 붕괴하지만(event 55 13.5, ADR 014
실측 1~7), 소프트 원판은 흐릿해도 그만큼 무너지지 않는다(event 50 무얼굴 옛날 사진 98.9). 그래서
**fallback 한정**으로 `whole_var < whole_image_collapse_variance`(기본 40 — 13.5의 3배, 98.9의
0.4배)면 쏠림 재확인을 면제하고 흔들림을 확정한다.

얼굴 경로에 면제를 두지 않는 이유: 블러 배경 프레임을 두른 옛날 사진(event 50의 7장)은 합성 블러
때문에 whole_var가 1.7~24.0까지 떨어져, 면제가 오탐을 그대로 되살린다. 얼굴 경로의 등방성 블러
미탐(face_var로는 흔들림 5.4 vs 옛날 사진 10.0~24.6이 겹쳐 붕괴 면제 불가)은 남은 한계로 문서화한다.

검증: event 55 해당 사진 blurry 복원 + 나머지 15장 무변경, event 50 오탐 8장 해제 유지,
test2 라벨셋 무회귀, `--smoke` 통과.

## 보강 2: 붕괴 면제의 얼굴 경로 확장 — 대형 주 인물 얼굴 한정 (2026-07-17, 당일)

보강 1이 fallback 한정이었던 구멍이 검출 강화와 겹치며 실제로 뚫렸다. event 64(그룹 27)에서
event 55와 동일한 고스팅 셀피(재업로드)가 이번엔 **얼굴 경로**로 빠졌다 — 대형 얼굴 회복 게이트
재보정(rel_w 0.30→0.20, 98a093c)으로 rel_w 0.28인 고스팅 얼굴이 검출되면서, face_var 10.6 < 25로
흔들림 후보였으나 쏠림 0.306 < 0.35로 게이트가 해제 → 쓰레기 임베딩 → HDBSCAN 노이즈 →
"분류가 어려워요" 앨범 유출. 보강 1 도입 시점엔 이 사진이 얼굴 미검출이라 fallback 면제로 잡혔다.

### 판별축 실측 (2026-07-17, 코퍼스 499장 — 로컬 라벨셋 + 캐시된 실 이벤트 24개)

게이트가 해제하는 부분집합(얼굴 경로 face_var<25 AND 쏠림<0.35) 안에서 고스팅 vs 옛날 인화:

| 후보 축 | 고스팅 | 옛날 인화 오탐 | 판정 |
|---|---|---|---|
| whole_var 단독 | 13.5 | 1.7~24.0 (+113.1) | ✗ 겹침 (보강 1과 동일) |
| face_var | 10.6 | 10.0~24.6 | ✗ 겹침 |
| rel_w 단독 | 0.280 | 최대 0.313 | ✗ 겹침 |
| 전체 쏠림 | 0.306 | 최대 0.268 | ✗ 마진 0.038 — 너무 얇음 |
| 자기상관 2차 피크 | 0.136 | 0.025~0.380 | ✗ 손떨림 궤적이 곡선이라 평행이동 피크 없음 |
| 타일 단위 쏠림 (med) | 0.501 | 0.44~0.92 | ✗ 인화지 테두리 직선 에지가 옛날 사진을 올림 |
| **rel_w AND whole_var 붕괴** | **0.280 & 13.5** | **붕괴 6장 rel_w≤0.172 / 대형 1장 whole_var 113.1** | **○ 양 갈래 분리** |

단일 축은 전부 기각이고 **결합 규칙이 성립한다**: 얼굴이 화면 대부분을 차지하면(rel_w↑)
whole_var는 사실상 얼굴 자체를 재는 것이라 fallback 붕괴 면제 논리가 그대로 이식되고, 옛날 인화
오탐은 (a) whole_var가 붕괴한 사진은 전부 얼굴이 작고(붕괴는 얼굴 밖 합성 블러 프레임의 가짜 신호)
(b) 얼굴이 큰 사진은 whole_var가 붕괴하지 않아 걸러진다. rel_w 조건의 필요성은 실 이벤트가 재확인
— event 16의 암실 전시 사진(어두워서 whole_var 12 붕괴, 배경 인물 rel_w 0.102)은 rel_w 조건이
없으면 새 오탐이 된다.

### 규칙과 채택값

```
얼굴 경로에서 blurry(variance 판정) AND blurry 얼굴 bbox 폭 ≥ 긴 변 × collapse_face_rel_width
AND whole_var < whole_image_collapse_variance  →  쏠림 재확인 면제 (흔들림 확정)
```

- **rel_w 하한 0.22**: 빈 구간 [0.172(옛날 인화 붕괴 최대), 0.280(고스팅)]의 기하 중앙(≈0.219).
  0.20도 코퍼스는 통과하나 오탐 측 마진이 1.16배로 얇다.
- **붕괴 임계는 `whole_image_collapse_variance`(40) 재사용**: 오탐 측 113.1(2.8배), 정탐 측
  13.5(3배)로 양쪽 마진 충분. 별도 임계를 늘리지 않는다.
- 구현: `quality.face_collapse_exempt()` + `judge_faces`가 blurry 얼굴 최대 폭을 함께 반환,
  `QualityConfig.collapse_face_rel_width` / `QUALITY_COLLAPSE_FACE_REL_WIDTH`, 0 = 비활성(보강 1 동작).
- 도구: `scripts/survey_face_collapse.py` (게이트 해제 부분집합의 신호·면제 발동 실측).

검증(프로덕션 배선 `build_face_extractor`, 검출 rel_w 0.20): 고스팅 얼굴 경로 blurry 복원 +
fallback 경로(검출 0.30) 유지, event 50 옛날 인화 59장 재점화 0, test2 라벨셋 무회귀(흔들림 6장
유지·선명 9장 통과), event 64 나머지 15장 무변경, 코퍼스 499장 스윕에서 면제 발동은 고스팅뿐,
`--smoke` 통과.

한계: 고스팅 **정탐 표본이 유니크 1장**(event 55·57·64·test5가 같은 파일의 재업로드 — md5 동일)
이라 정탐 측 임계는 단일 관측 기반이다. 미탐 리포트가 쌓이면 위 도구로 재실측한다.

## Consequences / 한계

- **등방성 블러 사각지대(소형 얼굴 한정으로 축소)**: 아웃포커스 주 인물, 고스팅·회전 손떨림
  얼굴(전역 쏠림 낮음)은 얼굴이 검출되면 variance가 잡아도 게이트가 해제한다. fallback 경로는
  variance 붕괴 면제(보강 1), 대형 주 인물 얼굴(rel_w ≥ 0.22)은 얼굴 경로 확장(보강 2)으로
  커버되지만, **작은 얼굴(rel_w < 0.22)은 원리적으로 남는다** — 작은 얼굴에서는 whole_var 붕괴가
  얼굴 상태의 증거가 못 되고(event 16 암실 사진이 반례), face_var는 옛날 인화와 겹친다(흔들림
  5.4 vs 옛날 사진 10.0). 실서비스 미탐 리포트가 쌓이면 라벨셋에 추가해 재보정한다.
- 블러 배경 프레임(앱 합성)은 등방 블러라 쏠림을 낮추는 방향으로만 작용 — 게이트와 정합적이다.
- 진짜 흔들린 옛날 재촬영 사진은 재촬영이 선명하면 원판의 흔들림 방향성이 보존돼 잡힐 수 있으나
  검증 표본이 없다.
