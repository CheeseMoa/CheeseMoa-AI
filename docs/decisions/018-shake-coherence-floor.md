# ADR 018: 흔들림 재확인 게이트 — variance 판정을 방향 쏠림 바닥으로 재확인

## Status

Accepted (2026-07-17). 흔들림 품질 게이트(CHMO-172)의 최종 게이트 추가 — 얼굴 경로·전체 이미지
fallback의 variance 기반 blurry 판정에 공통 적용. ADR 014의 방향 쏠림 신호를 반대 방향(해제)으로
재사용한다.

## Context

EC2 실서비스에서 event 50(그룹 27, 앨범 205)의 사진 8장이 전부 blurry로 오분류됐다. 로컬 재현
(59장)으로 확인한 결과 8장 모두 **옛날 인화 사진을 폰으로 재촬영하고 앱이 블러 배경 프레임을
씌운 PNG**였다: 7장은 얼굴 경로(정규화 variance 10.0~24.6 < 임계 25), 1장은 얼굴 미검출 fallback
(whole_var 98.9 < 임계 100)으로 탈락했다.

원인은 지표의 본질적 한계다. Laplacian variance는 **잔결(고주파 디테일)의 양**만 재므로,
"흔들려서 디테일이 뭉개진 사진"과 "원판 자체가 소프트해 디테일이 원래 없는 사진"(인화지 해상력
+ 재촬영 손실 + 앱 스무딩)을 구분하지 못한다. 임계 조정으로는 해결 불가 — event 50에서 통과한
옛날 사진들도 26.1, 34.5, 37.4…로 연속 분포라 빈 구간이 없고, 임계를 내리면 진짜 흔들림
(라벨셋 최대 22.3)을 놓친다.

### 판별 신호 실측 (2026-07-17, test2 라벨셋 + event 50 오탐 7장)

손떨림은 모든 에지가 한 방향으로 번지지만(ADR 014), 소프트 원판의 디테일 부재는 등방이다.
variance가 흔들림이라 한 얼굴들에서:

| 신호 | 흔들린 얼굴 (라벨셋) | 옛날 사진 오탐 얼굴 | 분리? |
|---|---|---|---|
| 얼굴 crop 방향 쏠림 | 0.132~0.349 | 0.044~0.454 | ✗ 겹침 |
| **전체 이미지 방향 쏠림** | **0.444~0.631** | **0.100~0.268** | **○ (빈 구간 0.268~0.444)** |

얼굴 crop 쏠림이 판별력이 없는 이유: 흔들린 얼굴 crop은 그라디언트 자체가 약해 구조 텐서가
crop 경계·노이즈에 지배된다. 전체 이미지 쏠림은 배경 에지까지 포함해 손떨림의 전역 방향성을
안정적으로 잡는다.

## Decision

variance 기반 blurry 판정(얼굴 경로 + fallback 1차 규칙)의 **최종 게이트**로 전체 이미지 방향
쏠림 바닥(`shake_coherence_floor`, 기본 0.35)을 추가한다 — `quality.shake_confirmed()`,
`deps.build_face_extractor`의 판정 말미에 적용:

```
blurry(variance 판정) AND coherence < floor  →  blurry 해제 (소프트 원판)
```

- **0.35 채택 근거**: 오탐 최고 0.268과 흔들림 최저(얼굴 미검출 fallback 라벨 0.397, 얼굴 경로
  0.444) 사이의 빈 구간에서 미탐 리스크가 적은 쪽(낮은 값)을 선택. ADR 014의 임계 0.40과 다른
  값인 이유: 그쪽은 "쏠림만으로 흔들림 확정"(높아야 안전), 이쪽은 "쏠림 부재로 흔들림 기각"
  (낮아야 안전)이라 보수 방향이 반대다.
- ADR 014의 2차 신호(쏠림 ≥ 0.40)로 잡힌 사진은 정의상 게이트(≥ 0.35)를 통과하므로 영향 없다.
- `QualityConfig.shake_coherence_floor` / `QUALITY_SHAKE_COHERENCE_FLOOR`, 0 = 비활성(기존 동작).

## 검증 (2026-07-17)

- **event 50 (59장)**: 오탐 8장 전부 해제(육안 검수 — 전부 비흔들림 옛날 사진), 나머지 51장 판정 무변경.
- **test2 라벨셋 회귀 없음**: 흔들림 6장 중 5장 유지(나머지 1장은 ADR 014에 문서화된 기존 미탐,
  쏠림 0.397 — 이 변경과 무관), 선명 9장 전부 통과 유지.
- `python -m app.worker --smoke` 8건 전부 통과.

## Consequences / 한계

- **등방성 블러 사각지대 확대**: 아웃포커스로 뭉개진 주 인물 얼굴, 회전 손떨림(전역 쏠림 낮음 —
  기존 문서화된 한계)은 variance가 잡아도 게이트가 해제한다. 현 라벨셋에 해당 표본이 없어 실측
  리스크는 미지수 — 실서비스 미탐 리포트가 쌓이면 라벨셋에 추가해 floor를 재보정한다.
- 블러 배경 프레임(앱 합성)은 등방 블러라 쏠림을 낮추는 방향으로만 작용 — 게이트와 정합적이다.
- 진짜 흔들린 옛날 재촬영 사진은 재촬영이 선명하면 원판의 흔들림 방향성이 보존돼 잡힐 수 있으나
  검증 표본이 없다.
