# ADR 016: 파편병합 face-level 응집 게이트 — centroid AND cross face-pair 평균

## Status

Accepted (2026-07-16, CHMO-269). [ADR 012](./012-merge-threshold-recalibration.md)가 "미검증 리스크"로
남긴 **아동 교차연령 오병합**을 실데이터로 확정·해소한다. ADR-012의 centroid 임계(0.55)는 유지하고,
그 위에 face-level 응집 조건을 AND로 추가하는 보강이다(임계 되돌림이 아님).

## Context

event 35(아이들 사진, 45얼굴/43사진)에서 `person1` 앨범에 **서로 다른 아이 30장**이 뭉쳤다. 층을
갈라 측정한 결과:

- **검출·HDBSCAN은 정상** — 생 HDBSCAN은 45얼굴을 12개 파편으로 잘 나눈다. 얼굴 크롭도 충분히 크다
  (bbox 폭 중앙값 398px, 112px 미만 4%뿐) → 해상도 문제 아님.
- **원인은 파편병합(`_merge_fragments`)** — 조각들의 **centroid**(정규화 평균) 코사인으로 병합하는데,
  어린아이 얼굴을 평균 내면 "아기 얼굴" 영역으로 수렴해 서로 다른 아이의 centroid도 0.55~0.63으로
  붙는다. 성인 타인 상한 0.4584(ADR-012)와 달리 아동 타인은 훨씬 가깝다 — 나이대 효과.
- **판별 신호는 개별 얼굴 쌍에 남아 있다** — centroid 평균이 지우지만, cross-파편 face-pair 유사도는:
  - 같은 인물 파편쌍: face 평균 **0.65** (face 최소 0.57)
  - 다른 아이 파편쌍: face 평균 **≤0.50** (face 최소 0.28~0.42)
  - → face 평균에 0.15 마진의 갭이 있다.

이는 단일 전역 centroid 임계로는 못 가른다: 아동 타인 centroid(≤0.63)와 성인 동일인 세션 간
centroid가 분포상 겹쳐, 임계를 올리면 성인이 재파편화되고 내리면 아동이 오병합된다.

### 측정 (라벨 코퍼스 ARI + 실 S3 이벤트, floor 스윕)

교정 후 파이프라인으로 라벨 코퍼스(`image/images/`: **child 8인 교차연령**·different 2인 성인·
단일인물 3셋)와 전 S3 이벤트 30개를 프로덕션 경로(recluster 전체 후처리)로 스윕:

| floor | child(8인) ARI | different(성인2인) | 단일인물 3셋 | 코퍼스 meanARI |
|---|---|---|---|---|
| 0.00 (현행) | **0.245** (4군집, 심한 과병합) | 1.000 | 무회귀 | 0.622 |
| 0.40 | 0.610 | 1.000 | 무회귀 | 0.805 |
| **0.45** | **0.788** (7군집) | 1.000 | 무회귀 | **0.894** |
| 0.475 | 0.784 (9군집) | 1.000 | 무회귀 | 0.892 |
| 0.50~0.55 | 0.637 (10군집, 과분할) | 1.000 | 무회귀 | 0.818 |

`child`만 floor에 반응해 상승하고, 성인·단일인물은 전 구간 floor=0 값 유지(무회귀). 0.45가 meanARI
고원의 정점이다. 0.50 이상은 아동을 과분할한다(8인을 10~11군집으로).

**실 S3 이벤트 적대적 검증** — 0.45에서 baseline(floor=0) 대비 분해되는 이벤트(27·29·33·35)가
오병합 수정인지 정상앨범 훼손인지, 갈라진 하위군집 쌍의 cross face-pair 평균으로 이벤트별 판정:
전부 **GOOD**(cross 평균 0.31~0.44 — 타인 분포, <0.50). event 35는 31장 blob이 [12,7,6,5,4,4]로
7명 분리, 최대 클러스터 내부 median 0.403→**0.592**, 타인 상한 미만 쌍 68.2%→**10.6%**. 회귀 0건.

## Decision

`ClusterConfig.merge_facepair_floor` 기본값 **0.45**. `_merge_fragments`의 병합 조건을 다음 AND로:

```
centroid_sim[i,j] >= merge_centroid_similarity  AND  facepair_mean[i,j] >= merge_facepair_floor
```

- `facepair_mean[i,j]` = 파편 i의 모든 얼굴 × 파편 j의 모든 얼굴 cross 코사인의 **평균**(단위벡터라 내적).
- candidate 필터와 complete-linkage 검사(`all(...)`) **두 곳 모두**에 AND 게이트. 1차·2차 병합 동일.
- **mean linkage 선택**: min(완전 연결)은 단일 하드 포즈 쌍에 취약하고 마진이 좁다(같은인물 min 0.57
  vs 다른아이 min 0.49, 갭 0.08). 평균은 갭 0.15로 더 안전하고 성인 포즈 변화에 강건하다.
- **0 = 비활성**: floor 0이면 항상 참 → centroid만으로 판정, 기존 동작과 완전 동일. `.env`
  `CLUSTER_MERGE_FACEPAIR_FLOOR`로 조정.
- 게이트는 병합을 **더 엄격하게만** 만든다 — centroid로 이미 붙던 쌍 중 face 응집이 낮은 것만
  추가로 막으므로, 새 오병합을 도입할 수 없다(안전 방향).

## 기각한 대안

- **centroid 임계만 상향(0.55→0.65)**: 아동 타인 centroid가 0.63까지 붙어 0.65에서도 과병합이 남고
  (스윕 확인), 성인 동일인 세션 간 쌍의 77%가 미달(ADR-012)이라 성인이 재파편화된다. 전역 단일
  임계로는 두 코호트를 못 가른다.
- **min linkage(완전 연결)**: 마진이 좁아(0.08) 성인 포즈 변화 시 정상 병합을 막을 위험. mean 채택.
- **임베딩 모델 교체 / 아동 전용 모델**: 라이선스로 봉쇄됨(CLAUDE.md). 후처리 게이트가 합법·저비용.

## 한계 · 리스크

- **floor 0.45는 이 코퍼스(child 8인 + 실 S3 30이벤트) 기준**이다. child 코퍼스가 8인 소규모라, 아동
  다수 이벤트 오병합 리포트가 새로 오면 스윕(`scripts/tune_merge_facepair.py`)부터 재실행할 것.
- event 35 최대 클러스터(12장)에 여전히 경계 얼굴이 섞일 수 있다(내부 min 0.346, 타인상한 미만 10.6%)
  — 완전 분리는 아니나 68.2%→10.6%로 대폭 개선. 더 조이면 아동 과분할(코퍼스 ARI 하락) 트레이드.
- **비용**: 파편 쌍마다 cross face-pair 행렬 계산이 추가되나 클러스터링은 전체 비용 0.1% 미만이라 무시
  가능. floor=0이면 계산 자체를 건너뛴다.

## Consequences

- event 35의 아동 30장 오병합이 기본값만으로 해소된다(7명 분리). 실 S3 이벤트 27·29·33도 오병합이
  함께 교정된다(적대적 검증 GOOD).
- ADR-012 §리스크(아동 교차연령 미검증)가 해소된다 — 다음 구현 목표 0.5의 트리거였던 실이벤트
  오병합 리포트에 대한 응답. centroid 임계 0.55는 그대로 유지된다.
- 이후 병합 관련 튜닝은 이 코퍼스 + 실 S3 이벤트 스윕(`tune_merge_facepair.py`)과 분해 판별
  (`verify_split.py`)을 재실행해 같은 방법으로 근거를 만들 것.
