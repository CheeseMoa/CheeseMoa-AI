# ADR 012: 병합 임계 재보정 0.68 → 0.55 — 입력 품질 교정 후 분포 측정 기반

## Status

Accepted (2026-07-15). [ADR 011](./011-same-photo-cannot-link.md)의 후속이자 일부 번복 — 같은 문제
(동일 인물 파편화)의 세 번째이자 결정적 지렛대. 전제 조건: 입력 품질 교정(정렬 안티에일리어싱 +
랜드마크 2단계 정제, 2026-07-15 배포분)이 적용된 파이프라인.

## Context

ADR-008~011은 병합 임계를 0.7 → 0.68로 소폭만 내리며 "0.6대 중반 금지"를 유지해 왔다. 근거는
**동일 인물 파편(최소 0.597)과 타인 centroid(최대 0.635)가 겹친다**는 실측이었다. 그러나 이
실측들은 전부 **입력 품질 교정 전** 임베딩으로 이루어졌다 — 당시 파이프라인은 같은 사진·같은
얼굴조차 처리 설정 하나에 유사도가 0.33까지 출렁이는 지터를 갖고 있었고([2026-07-14
리뷰](../reviews/2026-07-14-input-quality-alignment-landmark.md)), 그 지터 위에서 잰 분포 경계는
신뢰할 수 없었다.

교정 후 라벨 코퍼스(5인 33얼굴, 동일인 103쌍·타인 781쌍 — 닮은꼴 2인·마스크·저해상 단체사진
하드 케이스 포함)로 분포를 재측정한 결과([2026-07-15
리뷰](../reviews/2026-07-15-distance-distribution-verdict.md)):

- **타인 쌍은 최고 0.4584** — 781쌍 중 0.5 이상 0건. 닮은꼴 자매급(t4A×t4B)도 상한 0.2989.
- **동일 인물 세션 간 쌍(정상 포즈)의 77%가 0.68 미달** — 임계가 동일인 분포 깊숙이 박혀 정상적인
  표정·각도·조명 변화 대부분을 "남"으로 판정. 이것이 파편화의 직접 원인.
- 프로덕션 `recluster()` ARI 스윕: 0.68 → 0.6834(파편 3개), **0.45~0.60 전 구간 → 0.8118
  (5인 완벽 분리, 오병합 0)**, 0.40 → 첫 오병합(emb×karina 0.4124).
- ADR-009의 스윕 그리드는 {0.65, 0.70, 0.75}로 0.65 아래를 시험한 적이 없다 — "0.68 최적"은
  그리드 안에서의 최적이었을 뿐이다.

## Decision

`merge_centroid_similarity` 기본값 **0.68 → 0.55**.

- 0.55는 오병합 절벽(0.41)과 동일인 미병합 손실 사이 고원(0.45~0.60)의 안전 중앙 — 절벽까지
  마진 0.14, ADR-011의 하향 때 마진(0.045)의 3배.
- test4 실전(classify 전체 경로): 앨범 정확히 2개(A 8장·B 7장), 오병합 0. 구제·승격·같은사진
  cannot-link 등 기존 안전판과의 상호작용은 자가검증(cluster 20건·handlers 전 시나리오)으로 확인.
- ADR-011의 같은 사진 자동 cannot-link가 하향분의 오병합 노출을 구조적으로 줄인 상태에서의
  하향이다 (같은 사진에 공존하는 타인은 임계와 무관하게 병합 불가).

## 리스크 — 아동 교차연령 0.635는 재검증되지 않았다

ADR-008의 "타인 centroid 최대 0.635"는 face-test `child` 셋(8명 교차연령)에서 나온 값인데,
**교정 전 임베딩 기준**이고 해당 데이터가 현 개발 환경에 없어 교정 후 재측정을 못 했다. 이 값이
교정 후에도 유지된다면 0.55는 서로 다른 아동을 오병합할 수 있다.

- 완화: 같은 사진 자동 cannot-link(형제가 한 사진에 찍히는 흔한 케이스 차단), `.env`
  `CLUSTER_MERGE_CENTROID_SIMILARITY`로 코드 수정 없이 즉시 복귀 가능.
- **후속 조치**: face-test 데이터 확보 시 child 셋을 교정 후 파이프라인으로 재측정해 이 ADR을
  확정하거나 임계를 재조정할 것. 아동 다수 실 이벤트에서 오병합 리포트가 오면 그것이 트리거다.

## Consequences

- 파편화의 주 원인 제거 — 세션 간(헤어·조명·거리 변화) 동일 인물 파편이 병합 범위에 들어온다.
- 마스크·극단 옆모습(동일인인데 0.09~0.53) 얼굴은 여전히 안 붙는다 — 이는 임베딩의 진짜 한계로,
  uncertain 라우팅이 올바른 처리다. 임계로 해결하려 하지 말 것.
- InsightFace 유료 라이선스 검토는 근거 상실로 보류 — 무료 경로(입력 품질 교정 + 본 재보정)로
  test4가 완전히 풀렸다.
- ADR-009의 후속 스윕은 그리드를 0.45~0.75로 확장해 실행해야 한다 (기존 그리드는 하한이 높아
  이 재보정을 발견할 수 없었다).
