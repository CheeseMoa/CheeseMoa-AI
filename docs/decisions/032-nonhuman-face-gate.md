## 문제

인형·캐릭터 피규어가 사람으로 취급돼 **앨범이 만들어진다**(실 event 139의 `3e505676` = 몽치치 인형
3장만으로 구성된 앨범). 얼굴형 조형물은 같은 원인으로 **uncertain("분류가 어려워요")으로 샌다**.

이걸 거를 값싼 로컬 신호는 2026-07-20에 전부 실측 기각됐다 — 검출 score는 오히려 조형물이 더 높고
(0.89~0.92 vs 실얼굴 0.70~0.93), max_sim은 실인물 분포 한복판이며, 크기·종횡비는 정상값이다.
"우리 모델의 확신이 낮을 때만 외부에 묻는다"는 접근도 성립하지 않는다: **비인간 얼굴이야말로 검출
score가 높고 서로 잘 뭉쳐서 확신이 높게 나오기 때문**이다. 피해가 앨범 생성이므로 호출 지점은
"저신뢰 얼굴"이 아니라 **"앨범이 만들어지려는 순간"**이어야 한다.

## 결정

Rekognition `DetectLabels`·`DetectFaces`로 **얼굴 crop 단위 비인간 판정**을 하고, 비인간으로 확정된
얼굴을 재군집 입력에서 제외한다. 판정은 반드시 **2신호 AND**다.

### 판정 규칙 (실측 확정값 — 리뷰 §실측 ②)

```
crop = render_rejudge_crop(image, bbox)        # bbox+0.25 여백·q92·축소 없음 (ADR-030과 동일 파리티)

1콜: DetectLabels(crop, MinConfidence=50)
  ├ Sculpture | Statue ≥ 70                        → 규칙 B: 조형물 강등          (조형물 2/2, 오탐 0/55)
  ├ Doll | Toy ≥ 90 → 2콜: DetectFaces(crop)
  │                     └ FaceDetails 0개          → 규칙 A: 인형·그림 강등       (인형 3/3, 오탐 0/54)
  └ 그 외                                          → 통과 (현 동작 유지)
```

- **`DetectFaces` 미검출 단독 규칙은 기각**한다. 교실 단체사진의 실제 아이 17명 중 **4명(24%)이
  미검출**이었다(폭 33·33·37·43px, 육안 전수 확인 결과 전원 진짜 아이). 그 4명은 `Doll`/`Toy` 레이블이
  0건이라 규칙 A가 발동하지 않는다 — **AND가 오거부를 막는 유일한 장치다.**
- **조형물에는 `DetectFaces`를 요구하지 않는다.** 3D 사람형 조각은 Rekognition도 사람 얼굴로 검출한다
  (99.73·100.0). 규칙 B는 레이블 단독이며, 실인물 최고값 `Sculpture` 52.0(36px 아이) 대비 18p 여유다.
- **`Art`·`Person`은 쓰지 않는다.** 흐린 아이 얼굴이 `Art` 55.1·`Painting` 53.2를 받았고, 인형도
  `Person` 89.8~95.1·`Baby` 95.1을 받는다 — 둘 다 판별력이 없다.
- 판정 입력은 **타이트 crop 고정**이다. 넓은 crop은 인형 `Doll` 99.6→70.4, 조각 `Art` 100.0→56.0으로
  오히려 나빠지고, 사진 전체 호출은 인형 자리에 `Person`/`Baby` 인스턴스만 줘 실아동과 구분 불가다.
  crop 파라미터가 임계 90/70의 근거이므로 **파리티가 계약**이다(ADR-030 §crop과 같은 이유).

### 적용 지점 — 1차 패스 위에서 두 곳, 재군집 2차 패스는 1회

`_cluster_pass` 1차 결과에 대해 다음 두 대상만 판정한다(ADR-030 재판정 훅 **직전**에 위치):

1. **신규 앨범 후보**: 기존 cluster_id를 승계하지 않은 새 클러스터. 멤버 중 폭 내림차순 최대 3개를
   판정해 **과반이 비인간이면 클러스터 전체를 강등**한다. 대표 1장만 보지 않는 이유는 작고 흐린 실얼굴
   1장이 앨범을 통째로 날리는 사고를 막기 위해서다.
2. **미배정 얼굴**: uncertain 확정 직전의 노이즈+ambiguous 얼굴. 단건 판정(2026-07-20의 조형물 누출 경로).

**비인간 게이트를 재판정(ADR-030)보다 먼저 돌린다** — 강등된 얼굴을 재판정의 source·대표 후보에서
모두 배제해 CompareFaces 낭비를 없앤다. 실 event 139의 점수 캐시가 그 낭비를 이미 기록하고 있다:
인형(`f107bb79`)이 **앨범 대표로 뽑혀** 미배정 얼굴 4개와 비교됐고 4쌍 전부 `-1.0`(얼굴 미검출)으로
버려졌다 — 인형 앨범이 재판정의 비교 대상이 되는 경로가 실측으로 존재한다.
두 훅의 판정이 끝난 뒤 **2차 패스는 한 번만** 돈다(강등 기록 + soft-attach 기록 → `_cluster_pass` 1회).

### 강등의 의미 — 재군집 입력 제외 (스냅샷 직접 수정 기각)

강등은 `EventEmbeddings.nonhuman_face_ids`(신규, **npz v6**)에 face_id를 기록하는 것이고, 재군집은 그
행을 입력에서 제외한다. 그 결과 사진에 다른 얼굴이 없으면 "얼굴 0개" 경로를 타 **공용 앨범**으로
라우팅된다(feature-spec §6.2 — 원래 갔어야 할 곳).

스냅샷(클러스터 멤버 목록) 직접 수정은 ADR-030과 **같은 이유로 기각**한다: 저장된 `.npz`를 결정적
재군집한 결과와 발행 결과가 어긋나 재전달(at-least-once) 안전성이 깨진다. 기록 방식은 희소 튜플이라
`soft_attach_pairs`(v5)가 만든 선례를 그대로 따른다 — 고정 행 열 추가가 아니라 face_id 목록이다.

기록이 npz에 남으므로 **이후 어떤 재군집에서도 그 얼굴은 다시 앨범을 만들지 않고 재호출도 없다**
(ADR-030의 must-link 루프 차단과 같은 원리).

### 사람 결정 우선 — 면제와 해제

- **면제**: 얼굴 또는 그 클러스터 멤버가 사용자 보정(`must_link_pairs`·`cannot_link_pairs`)에 등장하면
  판정 자체를 생략한다. 사람이 "이 사람"이라고 확정한 것을 기계 증거가 뒤집으면 안 된다(ADR-030
  §3중 사전 검사 1과 같은 철학).
- **해제**: 강등된 얼굴에 사용자 보정이 들어오면(reassign 등으로 must-link 생성) `nonhuman_face_ids`
  에서 **제거**한다. 오판이 사용자 조작 한 번으로 영구 복구되는 경로가 반드시 있어야 한다.

### 판정 캐시 — `nonhuman-verdicts/{event_id}.json`

`face_id → {verdict, labels, n_faces}` 맵을 워커 소유 `embeddings_bucket`에 보관한다(ADR-030 점수
캐시와 같은 구도·같은 생명주기). **통과 판정도 캐싱해야** 매 재군집마다 같은 얼굴을 재호출하지 않는다
(강등분은 npz에 남지만 통과분은 안 남는다 — 캐시의 존재 이유가 여기다). 캐시는 소모품이라 손상 시
빈 캐시로 취급하고, 재군집 결정성의 **진실은 언제나 npz의 `nonhuman_face_ids`**다. 죽은 face_id는
저장 시 프루닝하고 event 전체 삭제 시 캐시 객체도 삭제한다.

### 안전판·격리 (ADR-030과 동일 정책)

- **best-effort**: 판정·캐시의 어떤 실패도 job을 죽이지 않는다. 실패 시 1차 패스 결과 그대로.
- **호출 상한** `NONHUMAN_MAX_CALLS_PER_JOB` — 초과분은 판정 보류 + `truncated` 로그(침묵 절단 금지).
- **롤백 스위치** `NONHUMAN_GATE_ENABLED=false`가 기본값이다. 표본이 얇아(비인간 4종) 실사용자 사진에
  기본 활성할 근거가 아직 없다 — 리뷰 §한계 참조.

## 활성화 게이트

ADR 030 §활성화 게이트(AWS AI 학습 opt-out·처리방침 위탁 명시·민감정보 동의)가 **그대로 적용**된다 —
같은 서비스라 새 법적 관문이 생기지는 않지만, 미해결분이 해결되지도 않는다. 추가로:

- 워커 IAM에 `rekognition:DetectFaces` · `rekognition:DetectLabels` 권한(없으면 best-effort 폴백으로
  종전 동작 + 경고 로그).
- 활성화 판단의 선행 조건은 **표본 축적**이다: 키노피오류 3D 플라스틱 피규어, 사실적 회화·인쇄물,
  마네킹. 리뷰 §한계의 미측정 항목이 닫히기 전에는 기본 false를 유지한다.

  `retired_cluster_ids` 계약과의 상호작용을 픽스처로 고정할 것.

## 한계

- **표본이 얇다**: 서로 다른 비인간 개체는 인형 1종·황금 조각 2개·애니 프린트 1장. event 96/139는
  같은 사진의 재업로드 미러라 독립 반복이 아니다. 임계 90/70은 이 표본 위의 값이다.
- **키노피오류 3D 캐릭터 피규어는 미측정** — 봉제 인형과 재질이 달라 조각처럼 검출될 수 있다.
- **소형 얼굴에서 규칙 A의 실질 방어는 `Doll|Toy ≥ 90` 하나**다. 얼굴이 작을수록 `DetectFaces` 미검출이
  흔해져 AND의 한쪽이 쉽게 참이 된다. 최소 얼굴 크기 조건 도입은 후속 판단.
- **사실적 회화·인쇄물(화장품 팔레트 그림, CLAUDE.md 목표 0.5)은 미검증** — 애니 프린트는 규칙 A로
  잡혔으나 사실적 그림에 `Doll`/`Toy`가 붙는다는 보장이 없다.
- 커버리지는 원본이 남은 얼굴뿐(bbox·s3_key 유효) — 구버전 `.npz` 행과 원본 소실분은 판정 불가.
- 강등은 face 단위이므로 **같은 사진에 실인물과 인형이 함께 있으면 실인물 앨범은 그대로** 유지된다.
  이는 의도된 동작이다.
