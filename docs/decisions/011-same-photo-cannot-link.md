# ADR 011: 같은 사진 자동 cannot-link + 병합 임계 0.70 → 0.68

## Status

Accepted (2026-07-14). [ADR 010](./010-post-rescue-second-merge.md)의 후속 — 같은 문제(동일 인물
파편화)의 두 번째 지렛대. [ADR 006/008](./008-blob-promotion-connected-components.md)의
"오병합보다 미병합" 원칙 위에 안전판을 먼저 깔고 임계를 소폭 내린다.

## Context

동일 인물의 세션 간(헤어·조명·해상도 변화) 임베딩 유사도(0.57~0.69)가 닮은 타인 상한(실측
0.635)과 겹쳐, 전역 병합 임계 하나로는 어디에 선을 그어도 한쪽이 틀린다. 실측에서 같은 인물
파편이 0.688~0.689로 구 임계 0.7에 반복적으로 아깝게 미달했다 (ADR-010의 event, 저해상도 실험).

한편 **같은 사진에 공존하는 두 얼굴은 100% 서로 다른 사람**이라는, 임베딩보다 강한 사실 신호가
있다. 기존에는 "단일 사진 클러스터 강등"(handlers)에만 소극적으로 썼는데, 실 event 전수 검사에서
**한 사진의 얼굴 4개(상호 유사도 0.29~0.41)가 한 클러스터에 저장된 라이브 오분류**가 발견됐다
(event 13, c200e6fb) — 다중 사진 클러스터 안의 같은사진 공존은 강등 규칙이 못 잡는다.

## Decision

1. **`Constraints.auto_cannot_link` 신설** — 같은 사진 얼굴 행 전 쌍을 handlers가 photo_ids에서
   매 재군집마다 유도해 전달한다 (`_same_photo_cannot_links`, 저장 안 함). 사람 cannot-link와
   동일하게 참여: 클러스터 분리(enforce), 파편 병합 차단(1·2차), 노이즈 구제 차단, must-link
   대상 라벨 제외.
2. **사람 제약 특혜에는 불참** — 저신뢰 축출 보호와 축출 마진 비교 제외는 사람 cannot-link만.
3. **must-link 모순 자동 쌍은 탈락** (거부 아님) — 사람 결정 우선. 동일 사진 중복 업로드 오염
   같은 예외에서 재군집이 ValueError로 죽지 않아야 한다.
4. **이중 검출 안전판** — 같은 사진 쌍의 임베딩 유사도 ≥ 0.95면 YuNet이 한 얼굴을 두 박스로
   검출한 것으로 보고 제약에서 제외 (실측 같은사진 타인 쌍 최대 0.61, 근중복 0.985와 큰 마진).
5. **`merge_centroid_similarity` 0.70 → 0.68** — 실측 미달 사례(0.688·0.689)를 회복하는 최소
   하향. 타인 상한 0.635 대비 마진 0.045를 남긴다 (0.6대 중반 하향은 ADR-008·009가 기각한 그대로 금지).

## 왜 축출 특혜에서 빼는가 (시뮬레이션 발견)

자동 쌍을 기존 cannot-link 채널에 그대로 넣은 시뮬레이션에서, 다인 사진이 많은 event는 사실상
전 얼굴이 "제약 당사자 = 축출 보호" 대상이 됐다. HDBSCAN은 밀도 구조가 없는 데이터에서도
무의미한 클러스터(쌍 유사도 0.03~0.28의 남남 묶음)를 내는데, 평소엔 저신뢰 축출(LOO < 0.4)이
전부 걸러낸다 — 보호가 전면화되면 이 방벽이 무력화되어 남남 앨범 4개가 살아나는 것이 실 event
(17번)에서 재현됐다. 축출 보호의 근거는 "사람이 직접 지목한 얼굴"이라는 신뢰이므로, 기계 유도
제약은 받을 자격이 없다. `cluster.py` 자가검증 (k)가 이 대비(사람 쌍은 보호, auto 쌍은 정상 축출)를
회귀 고정한다.

## 검증

- S3 실 event 17개 전수 재시뮬레이션: 오병합 0건. event 13의 같은사진 4얼굴 오분류 해체,
  event 16 파편 병합 유지, event 17 무의미 클러스터 0개 유지(축출 정상 동작).
- 저해상도 스트레스(스크린샷 9장, 2인): 임계 0.68로 세션 파편(0.689) 회복, 같은사진 가드로
  두 인물 간 오병합 0건 유지.
- 정답 라벨 하에서 0.60까지 내려도 오병합이 없었으나, 닮은 타인이 공존 사진 없이 나타나는
  케이스(별도 촬영된 형제 등)는 미검증이라 보류 — 라벨 데이터 확보 후 재검토 (2단계).

## Consequences

- 자동 제약은 `.npz`에 저장하지 않는다 — photo_ids에서 항상 유도 가능하고, 저장하면 행
  추가/삭제 마스킹과의 정합 관리가 생긴다.
- 다인 사진이 많은 event는 제약 쌍 수가 늘지만(사진당 C(n,2)), enforce/merge/rescue의 쌍 검사
  비용은 클러스터링 전체가 파이프라인 비용의 0.1% 미만이라 무시 가능하다.
- 같은 사진 얼굴이 같은 클러스터에 저장되는 상태는 이제 구조적으로 불가능하다 (이중 검출
  안전판에 걸리는 근중복 쌍 제외).
