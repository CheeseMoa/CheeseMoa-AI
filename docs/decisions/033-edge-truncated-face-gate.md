# ADR 033: 프레임 밖 잘린 얼굴 게이트 — 눈 랜드마크 이탈 판정

- 상태: **제안 (초안, 2026-07-24)** — 미구현
- 관련: [2026-07-24 실측 스윕](../reviews/2026-07-24-edge-truncated-face-sweep.md)(근거 데이터 전체) ·
  ADR 013/015/028(기존 검출 방어층 — 전부 미발동임을 실측) · ADR 023(정제·재검출 랜드마크 신뢰 —
  판정 시점이 여기에 걸린다) · [ADR 032](032-nonhuman-face-gate.md)(비인간 얼굴 게이트 — **이 문제와
  다른 축**) · ADR 007(재군집 흐름 — 기존 npz 행을 못 고치는 이유) · CLAUDE.md §다음 구현 목표 0.4

## 문제

event 144에서 사진 한 장이 "분류가 어려워요"(uncertain)로 샜다. 원인은 사진 오른쪽 위, **프레임
경계에서 잘린 반쪽 얼굴**이다(인화 사진 판을 다시 찍은 사진의 옆 사진 얼굴).

YuNet은 얼굴을 검출하면 랜드마크 5점을 **항상** 뱉는다 — 프레임 밖 부위도 좌표를 추측해서 찍는다.
이 얼굴은 5점 중 3점(왼눈·코·왼입)이 이미지 폭 3024를 넘은 3041~3043에 찍혔다. 정렬(Umeyama)은 그
추측 점들로 계산되므로 **임베딩이 원리적으로 성립하지 않고**, 실제로 event 전체 최근접 유사도가
0.234에 그쳐 아무 군집에도 안 붙었다. 그런데 폭 345px가 그 사진 최대 얼굴 폭(647px)의 50% 게이트를
넘어 **주 인물 자격**을 얻는 바람에, 사진이 uncertain으로 라우팅됐다.

기존 방어층은 전부 미발동이다(실측 리뷰 §배경): score 0.866이 높아 ADR 015의 결합 필터(score<0.78
**AND** 종횡비<0.70)가 안 걸리고, rel_w 8.6%라 ADR 028의 대형 게이트 대상도 아니다.

**ADR 032(비인간 얼굴 게이트)로는 못 잡는다.** 실측으로 확인했다 — `Doll`/`Toy`·`Sculpture`/`Statue`
라벨이 0건이고 `DetectFaces`는 얼굴을 정상 검출한다(87.49). 당연한 결과다: **이건 진짜 사람의 반쪽
얼굴이라 "비인간이냐"는 질문 자체가 빗나간다.** 두 ADR은 이름이 비슷할 뿐 다른 축의 문제다.

## 결정

검출 파이프라인 마지막에서 **눈 랜드마크가 이미지 밖을 가리키는 얼굴을 폐기**한다.

```
faces = [정제·디둡·회복까지 끝난 최종 얼굴들]           # detect() 반환 직전
for face in faces:
  (ex0, ey0), (ex1, ey1) = face.landmarks[0], face.landmarks[1]   # ArcFace 순서: 0·1이 양눈
  if 두 눈 중 하나라도 [0, w-1] × [0, h-1] 밖  →  폐기
```

### 왜 "눈"인가 — 개수가 아니라 부위다 (실측 §③)

전수 스윕(사진 609장·얼굴 1182개)에서 랜드마크가 밖으로 나간 얼굴은 35행/**고유 5종**이고, 부위로
가르면 경계가 선명하다:

| 밖으로 나간 부위 | 고유 얼굴 | 앨범 배정 |
|---|---|---|
| **눈 포함** | 3종 (반쪽 얼굴 · 턱목 조각 · 눈 잘린 남자) | **0 / 18행** |
| 코·입만 | 2종 (옆면만 걸친 여자아이 · 뒤통수) | **5 / 17행** (여자아이가 5개 이벤트에서 정상 멤버) |

- **"랜드마크 1개라도 밖이면 폐기"는 기각한다** — 오른쪽 끝에 옆면만 살짝 잘린 여자아이(왼입만 밖,
  종횡비 0.62, score 0.908)를 죽인다. 실제로 5개 이벤트에서 인물 앨범에 잘 붙어 있는 얼굴이다.
- **"밖으로 나간 거리"도 기각한다** — bbox가 경계에서 클램프되니 추측 점도 경계 바로 바깥에 찍힌다
  (out_ratio p10 0.01 ~ p99 0.09). 거리로는 정상 얼굴과 안 갈린다.
- 눈이 판별선인 것은 우연이 아니라 **정렬의 불변식**이다: Umeyama 변환의 스케일·회전을 지배하는 것이
  `_ARCFACE_DST`의 양눈 두 점이라, 눈이 추측값이면 정렬 결과 전체가 무효다. 코·입만 추측이면 정렬은
  견딘다 — 위 표가 그 예측과 정확히 일치한다.

이 코퍼스에서 "눈 밖"과 "밖 랜드마크 2개 이상"은 같은 분할을 만들지만, **채택하는 규칙은 눈**이다.
개수는 상관관계고 눈은 인과다.

### 적용 지점 — `detect()` 반환 직전 (정제·회복 이후)

`_refine_face`(2단계 랜드마크 정제)·`_recover_large_face`(ADR 017 회복)·`_dedup_confident_fragments`가
**끝난 뒤**의 최종 랜드마크로 판정한다. 정제·재검출이 랜드마크를 교정하므로(ADR 023: 재검출 score≥0.80
이면 이동 가드를 무시하고 재검출 랜드마크를 채택), 그 전에 판정하면 교정으로 프레임 안에 들어올 얼굴을
먼저 죽인다.

### 폐기의 의미 — 그 얼굴은 없던 것이 된다

폐기된 얼굴은 임베딩·npz 행·머릿수·주 인물 판정 어디에도 등장하지 않는다. 별도 기록 열이 필요 없다
(ADR 032의 `nonhuman_face_ids`와 다른 점 — 저쪽은 이미 저장된 행의 재군집 참여를 막는 것이라 기록이
필요하지만, 이쪽은 검출 단계라 애초에 행이 안 생긴다).

사진에 다른 얼굴이 없으면 "얼굴 0개" 경로를 타 **공용 앨범**으로 간다(feature-spec §6.2 — 원래 갔어야
할 곳). event 144의 사진은 정상 얼굴 2개가 남아 인물 앨범 2개 + 공용 노출이 그대로 유지되고,
uncertain에서만 빠진다.

### 기본값 — 활성

`DetectorConfig.truncated_eye_gate: bool = True`(롤백 시 `False`).

ADR 032가 기본 비활성을 택한 것과 갈리는 이유는 신호의 성격이다: 저쪽은 외부 모델의 확률적 라벨에
임계를 걸어 **진짜 아이를 오거부할 위험**이 실측으로 존재하지만(교실 단체사진 4/17), 이쪽은
**"얼굴 정렬의 기준점이 사진에 찍히지 않았다"는 기하학적 사실**이고 관측 오거부가 0이다. 무과금·
결정적이라 재현·회귀 고정도 쉽다.

## 기각한 대안

| 대안 | 기각 사유 |
|---|---|
| 랜드마크 1개라도 밖이면 폐기 | 정상 앨범 멤버(옆면만 잘린 여자아이) 오거부 — 실측 5/17행 |
| 밖으로 나간 거리(out_ratio) 임계 | 판별력 없음 — 정상·비정상 모두 0.01~0.09 |
| bbox가 경계에 닿으면 폐기 | 경계에 닿아도 멀쩡한 얼굴이 대부분 — 위 여자아이·뒤통수 포함 전부 폐기됨 |
| 종횡비 임계 강화(ADR 015 완화) | 이 얼굴은 종횡비 0.50이지만 score 0.866 — score 조건을 풀면 ADR 028이 막아둔 손·종이 오검출이 되돌아온다 |
| Rekognition `DetectLabels`(ADR 032 규칙) | 미발동 실측 — 진짜 사람이라 비인간 신호가 없다 |
| Rekognition `Art`/`Painting` ≥ 90 | ADR 032가 이미 기각한 축이고, 진짜 아이가 `Art` 84.9·`Painting` 57.0을 받은 실측이 있다. 얼굴당 유료 호출을 무료 로컬 판정으로 대체 가능 |
| uncertain 라우팅에서만 제외(주 인물 자격 박탈) | 증상만 가린다 — 추측 랜드마크 임베딩이 남 사람 앨범에 붙거나 유령 앨범을 만드는 경로가 그대로다 |

## 검증 (구현 시 자가검증 계획)

- `python -m app.pipeline.detect` — ⓐ 합성 이미지로 눈 좌표가 경계 밖/안 경계값(w-1, w) ⓑ 코·입만 밖인
  얼굴은 유지 ⓒ 게이트 off면 종전 동작 ⓓ 정제·회복으로 눈이 프레임 안으로 교정되는 얼굴은 유지
  (판정 시점이 정제 이후임을 고정)
- `python -m app.handlers` — 종단: 잘린 얼굴만 있는 사진이 uncertain이 아니라 공용 앨범으로 · 잘린
  얼굴 + 정상 얼굴 사진은 인물 앨범 유지 + uncertain 미등장
- `python -m app.worker --smoke` — 발행 경유 왕복
- 회귀 코퍼스: `python scripts/survey_edge_truncated_faces.py` 재실행 — 눈 밖 3종이 사라지고 코·입만
  밖인 2종(특히 앨범 멤버 여자아이)이 그대로 남는지

## 한계

- **고유 표본이 얇다**: 잘린 얼굴 고유 5종(눈 밖 3종·눈 안 2종)이다. 스윕의 35행은 같은 사진의
  재업로드 미러 배수라 독립 반복이 아니다.
- **효과 범위가 좁다**: 주 인물 미배정 얼굴을 가진 사진 134장(609장 중) 가운데 이 게이트가 건드리는
  것은 고유 1장이다. uncertain의 대부분은 잘림과 무관하다(미배정 420개가 밖 랜드마크 0개).
- **기존 이벤트는 안 고쳐진다**: 재군집은 저장된 `.npz`로 돌고 재검출을 하지 않으므로(ADR 007), 이미
  저장된 잘린 얼굴 행은 그대로 남는다 — event 144를 재분류해도 그 사진은 계속 uncertain이다. npz에
  랜드마크를 보관하지 않아 사후 판정도 불가하다. **신규 업로드분부터 적용**이며, 소급이 필요하면
  event 삭제 후 재업로드가 유일한 경로다.
- **눈이 안에 있는데 얼굴 절반이 밖인 경우**(정면 얼굴의 좌우 절반만 걸침)는 통과한다 — 이번 코퍼스에
  표본이 없어 미측정이다.
- **가려진 얼굴은 대상이 아니다**: 다른 물체·사람에 가려진 얼굴은 랜드마크가 이미지 안에 있어 이
  신호로 안 잡힌다([2026-07-23 가림·포즈 조사](../reviews/2026-07-23-occlusion-pose-split-survey.md)의 별건).
- **극단 회전**(누운 얼굴이 경계에 걸린 경우)에서 눈이 밖으로 나가는 정상 얼굴이 있을 수 있다 —
  관측된 적은 없다.
