# ADR 008: 소규모 단일 인물 이벤트 교정 — 연결 성분 부분 승격 + 파편 병합 임계 완화

## Status

Accepted — [ADR 006](./006-postprocessing-accuracy-hardening.md) 결정 3(균질 blob 승격 조건)을 대체한다

## Context

[ADR 006](./006-postprocessing-accuracy-hardening.md) 결정 3은 HDBSCAN이 클러스터 0개를 내는
퇴화(`allow_single_cluster=False`에서 분할 지점이 없는 균질 event)를 "전원 노이즈 + **전 쌍별
유사도 ≥ merge_centroid_similarity(0.7)** 일 때만 전체를 단일 클러스터로 일괄 승격"으로 교정했다.
당시 검증은 합성 데이터(근중복 버스트)였다.

이후 face-test 실사진 검증(181장)에서 한계가 확인됐다:

- **다인물 이벤트는 순도 100%**(오병합 0건)로 분리에 성공했으나,
- **동일 인물 4~6장뿐인 소규모 이벤트는 인물 앨범이 아예 생기지 않았다** — 같은 5장을 타인과
  섞으면 정확히 묶이는데, 단독이면 전원 `uncertain(unmatched)`이 된다.
- 원인: 포즈 변화가 있는 실사진의 **동일 인물 쌍 유사도는 0.46~0.70**이라(타인 ≲0.3), 한 쌍이라도
  0.7 미만이면 `gram.min() ≥ 0.7` 완전 연결 조건이 깨져 승격이 통째로 실패한다. 0.7 일괄 승격은
  사실상 근중복 버스트만 구제한다. 승격이 실패하면 클러스터가 0개라 파편 병합·노이즈 구제도
  전부 무력하다 (붙일 클러스터 자체가 없음).

추가 결함: 일괄 승격(`labels[:] = 0`)은 all-or-nothing이라 "단일 인물 N장 + 낯선 행인 1장"
이벤트(이 경우도 condensed tree에 진짜 분할이 없어 클러스터 0개가 된다)에서 조건을 낮추면
행인까지 오병합하고, 낮추지 않으면 전체가 기각된다 — 어느 쪽도 옳지 않다.

소규모 단일 인물 이벤트는 클러스터 0개만이 아니라 ADR 005의 다른 갈래인 **파편화**로도 깨진다.
실사진 재검증(train 8인물 × 4~5장 단독 이벤트)에서 동일 인물 5장이 HDBSCAN `[3, 2]` 파편이 되고,
파편 간 centroid 유사도 **0.68**이 병합 임계 0.7에 아슬하게 미달해 같은 사람의 앨범이 2개로
갈라지는 사례(1/8건)를 확인했다 — 임계 0.7은 개별 쌍 유사도 분포(동일 인물 ≳0.7) 가정에서 왔지만,
파편 centroid 간 유사도는 그보다 낮게 형성된다.

## Decision

`_promote_single_blob`을 **연결 성분 기반 부분 승격**으로 재설계한다. 전원 노이즈일 때:

1. 쌍 유사도 ≥ `blob_promote_similarity`(기본 **0.45**)인 얼굴끼리 간선으로 이어 연결 성분을 만든다.
2. 다음을 모두 만족하는 성분을 **각각** 클러스터로 승격한다 (최대 성분만이 아님):
   - 크기 ≥ `min_cluster_size` — 단독 행인·짝 없는 얼굴은 노이즈 유지
   - 성분 내 **모든 쌍** ≥ `blob_promote_floor`(기본 **0.4**, 완전 연결) — 닮은 중간자 체이닝
     (A~B, B~C인데 A·C는 남남)이 두 인물을 한 성분으로 잇는 오병합 차단
3. 조건 미달 성분은 통째로 기각한다 (오병합보다 미병합 — ADR 006 원칙 유지).

두 임계값은 `ClusterConfig` 설정값이며(환경변수 `CLUSTER_BLOB_PROMOTE_*`), 교차 불변식
`min_membership_similarity ≤ blob_promote_floor ≤ blob_promote_similarity`를 생성 시점에 강제한다.
"전원 노이즈일 때만 동작" 전제와 "승격 → 보정 제약 강제" 순서는 유지한다.

함께, 파편화 갈래의 교정으로 **파편 병합 임계 `merge_centroid_similarity` 기본값을 0.7 → 0.6으로
완화**한다 (ADR 006 결정 2의 완전 연결 메커니즘은 그대로, 임계값만 실데이터로 재조정 — 원 주석에
예고된 "TBD #4 실데이터 재조정"의 수행).

## Rationale

- **대안 (a) — 일괄 승격 임계만 완화(0.7 → 0.45)**: 기각. 순수 단일 인물 이벤트는 구제하지만
  all-or-nothing이라 "인물 + 행인" 이벤트에서 여전히 전체 기각(앨범 미생성)되거나 행인을 오병합한다.
- **대안 (b) — `allow_single_cluster=True`**: 기각 유지. 단일 클러스터 모드의 루트 소속 판정이
  `cluster_selection_epsilon=0.15`(유사도 0.85 이내)를 요구해 실사진 분산(~0.5~0.7)이 오히려
  전원 노이즈가 됨을 실험으로 확인했다 ([ADR 005](./005-hdbscan-standalone-port.md)).
- **대안 (c) — HDBSCAN 파라미터 재조정**: 기각 유지. PoC 검증 레시피를 흔들며, 다인물 순도
  100%를 담보하는 현 파라미터를 건드릴 근거가 없다 (TBD #4 실데이터 튜닝의 몫).
- **선택 — 연결 성분 + 완전 연결 floor의 2단 구조**: 간선(0.45)은 실측 동일 인물 하한(0.46)
  직하에서 소속을 판정하고, floor(0.4)는 성분 내부의 완전 연결을 보증해 체이닝 오병합을 차단한다
  (타인 ≲0.3 < 0.4). 부분 승격이라 행인은 자연히 노이즈로 남고, 이후 노이즈 구제(0.6)가 신규
  클러스터 centroid 기준으로 재시도한다 — 기존 후처리 흐름을 그대로 재사용한다.

**기본값 근거** (실측 분포: 동일 인물 쌍 0.46~0.70, 타인 ≲0.3):

| 필드 | 기본값 | 근거 |
|------|--------|------|
| `blob_promote_similarity` | 0.45 | 동일 인물 하한 0.46 직하 (실사진 포섭), 타인 상한 대비 +0.15 마진 |
| `blob_promote_floor` | 0.4 | 타인 상한 +0.1 (체이닝 차단). `min_membership_similarity`(0.4)와 같아 승격 멤버의 LOO 유사도 ≥ floor가 보장되어(‖타멤버 합‖ ≤ n−1) 같은 실행의 저신뢰 축출 재강등이 없다 |

**병합 임계 0.6의 실측 근거** (face-test train, 2026-07-04 재검증):
- 단일 인물 4~5장 단독 이벤트 8건: 0.7에서는 1건이 `[2, 2] + ambiguous 1`로 파편화(파편 centroid
  0.68), 0.6에서는 8건 전부 앨범 1개로 완치. 나머지 7건은 두 임계에서 결과 동일.
- 다인물 혼합: 6인물 28얼굴·**20인물 180얼굴** 모두 0.6에서 오병합 0건, 0.7과 클러스터 구성 동일.
  타인 파편 간 centroid 유사도 실측 최대 **0.167** — 임계 0.6 대비 3배 이상의 마진.
- 노이즈 구제 임계(`rescue_similarity=0.6`, "동일 인물 하한 ≈0.6")와도 정합한다.

**다인물 순도가 깨지지 않는 이유**: 부분 승격은 전원 노이즈(클러스터 0개)일 때만 동작한다 —
HDBSCAN이 클러스터를 하나라도 만든 보통의 다인물 이벤트에는 무접촉이다. 병합 완화는 위 실측대로
타인 centroid 분포(≲0.17)와 3배 마진이 있고 완전 연결 검사가 이중 방벽이다. 승격 뒤에도
cannot-link 강제·저신뢰 축출이 기존 순서대로 적용된다.

## Consequences

- 소규모 단일 인물 이벤트(4~6장, 실측 유사도 대역)에서 인물 앨범이 생성된다 — 클러스터 0개
  갈래는 부분 승격이, 파편화 갈래는 병합 임계 0.6이 교정한다 (실사진 8건 전부 앨범 1개 확인).
  동일 인물 2장 이벤트(쌍 ≥0.45)도 구제 범위에 들어온다 (기존: 쌍 ≥0.7 근중복만).
- "단일 인물 + 행인" 클러스터 0개 이벤트에서 행인만 노이즈로 남는다 (일괄 승격으로는 불가능).
- 구 0.7 완전 연결 fast-path는 제거 — `gram.min() ≥ 0.7`이면 새 로직이 정확히 같은 결과를 내고
  (전체가 한 성분 + floor 통과), 쌍별 임계에 centroid 병합 임계(`merge_centroid_similarity`)를
  재사용하던 결합이 끊어진다. 근중복 버스트 동작 보존은 합성 회귀로 고정했다.
- **의도된 한계 (fail-safe)**: 성분 검사는 all-or-nothing이다 — "인물 4장 + 어중간한 닮은꼴
  행인(한 멤버와만 0.45+, 나머지와 <0.4)"이면 성분 통째 기각으로 앨범이 안 생긴다. AuraFace
  분포상 타인 0.45+ 쌍은 희귀해 MVP는 수용한다. 실데이터에서 재현되면 "최저 행합 노드를 하나씩
  제거(peel)하며 재검사"하는 구제를 후속 검토한다.
- **알려진 한계**: 교차 쌍이 전면 0.4+인 진짜 닮은 두 인물(쌍둥이급)은 한 성분으로 승격될 수
  있다 — 유사도 기반 방법의 공통 한계이며 사용자 보정(cannot-link)이 교정 수단이다.
- 회귀 고정: `cluster.py __main__` 합성 스위트 12건(실측 대역 5장·행인 혼입·두 인물 분리·
  근중복 버스트·체이닝 기각·불변식) + `handlers.py __main__` 종단 케이스 1건(단일 인물 5장
  classify → 앨범 1개·uncertain 0건). 각 e2e 케이스는 HDBSCAN이 실제로 클러스터 0개를 내는지
  전제부터 검증한다.
- 실사진 검증 완료 (로컬 `scripts/classify_local.py` + face-test train): 다인물 20인물 180얼굴
  오병합 0건·앨범 20/20 생성, 병합 임계 0.7↔0.6 간 클러스터 구성 동일 (순도 100% 유지).
