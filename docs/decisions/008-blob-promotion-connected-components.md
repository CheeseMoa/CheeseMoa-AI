# ADR 008: 소규모 단일 인물 이벤트 교정 — 연결 성분 부분 승격(+peel)

## Status

Accepted — [ADR 006](./006-postprocessing-accuracy-hardening.md) 결정 3(균질 blob 승격 조건)을 대체한다

## Context

[ADR 006](./006-postprocessing-accuracy-hardening.md) 결정 3은 HDBSCAN이 클러스터 0개를 내는
퇴화(`allow_single_cluster=False`에서 분할 지점이 없는 균질 event)를 "전원 노이즈 + **전 쌍별
유사도 ≥ merge_centroid_similarity(0.7)** 일 때만 전체를 단일 클러스터로 일괄 승격"으로 교정했다.
당시 검증은 합성 데이터(근중복 버스트)였다.

이후 face-test 실사진 검증에서 한계가 확인됐다:

- **다인물 이벤트는 순도 100%**(오병합 0건)로 분리에 성공했으나,
- **동일 인물 4~6장뿐인 소규모 이벤트는 인물 앨범이 아예 생기지 않았다** — 같은 5장을 타인과
  섞으면 정확히 묶이는데, 단독이면 전원 `uncertain(unmatched)`이 된다.
- 원인: 포즈 변화가 있는 실사진의 **동일 인물 쌍 유사도는 0.46~0.70**이라(타인 ≲0.3), 한 쌍이라도
  0.7 미만이면 `gram.min() ≥ 0.7` 완전 연결 조건이 깨져 승격이 통째로 실패한다. 0.7 일괄 승격은
  사실상 근중복 버스트만 구제한다. 승격이 실패하면 클러스터가 0개라 파편 병합·노이즈 구제도
  전부 무력하다 (붙일 클러스터 자체가 없음).

일괄 승격(`labels[:] = 0`)은 all-or-nothing이라 "단일 인물 N장 + 낯선 행인 1장" 이벤트(이 경우도
클러스터 0개가 된다)에서 조건을 낮추면 행인까지 오병합하고, 낮추지 않으면 전체가 기각된다. 또한
실사진 karina 5장은 한 쌍(karina4↔karina5)이 0.394로 완전 연결 floor를 아슬하게 깨, **한 장 때문에
성분 전체가 기각**되는 것이 재현됐다.

## Decision

`_promote_single_blob`을 **연결 성분 기반 부분 승격 + peel**로 재설계한다. 전원 노이즈일 때:

1. 쌍 유사도 ≥ `blob_promote_similarity`(기본 **0.45**)인 얼굴끼리 간선으로 이어 연결 성분을 만든다.
2. 각 성분에서 **모든 쌍 ≥ `blob_promote_floor`(기본 0.4, 완전 연결)** 인 최대 부분집합(크기 ≥
   `min_cluster_size`)을 각각 클러스터로 승격한다. 완전 연결이 깨지면 성분을 통째 버리지 않고,
   **연결이 가장 약한 멤버(성분 내 유사도 합 최소)를 하나씩 떼며 재검사한다(peel)**.
3. floor는 닮은 중간자 체이닝(A~B, B~C인데 A·C는 남남)이 두 인물을 한 성분으로 잇는 오병합을
   차단한다 (타인 쌍 ≲0.3 < floor).

두 임계값은 `ClusterConfig` 설정값이며(환경변수 `CLUSTER_BLOB_PROMOTE_*`), 교차 불변식
`min_membership_similarity ≤ blob_promote_floor ≤ blob_promote_similarity`를 생성 시점에 강제한다.
"전원 노이즈일 때만 동작" 전제와 "승격 → 보정 제약 강제" 순서는 유지한다.

**파편 병합 임계 `merge_centroid_similarity`는 PoC 검증값 0.7로 유지한다** (0.6 완화안은 아래 근거로 기각).

## Rationale

- **대안 (a) — 일괄 승격 임계만 완화(0.7 → 0.45)**: 기각. 순수 단일 인물 이벤트는 구제하지만
  all-or-nothing이라 "인물 + 행인" 이벤트에서 여전히 전체 기각(앨범 미생성)되거나 행인을 오병합한다.
- **대안 (b) — `allow_single_cluster=True`**: 기각 유지. 단일 클러스터 모드의 루트 소속 판정이
  `cluster_selection_epsilon=0.15`(유사도 0.85 이내)를 요구해 실사진 분산(~0.5~0.7)이 오히려
  전원 노이즈가 됨을 실험으로 확인했다 ([ADR 005](./005-hdbscan-standalone-port.md)).
- **대안 (c) — 파편 병합 임계 0.7 → 0.6 완화**: **기각.** 파편화 갈래(동일 인물이 HDBSCAN
  `[3, 2]` 파편이 되고 파편 centroid ~0.68이 0.7에 미달해 앨범 2개로 갈림)를 노려 0.6을 검토했으나,
  centroid 유사도가 겹쳐 전역 임계로 분리 불가함이 실측으로 드러났다 (face-test train):
  | 분포 | centroid 유사도 |
  |------|------|
  | 서로 다른 아동(교차연령) | 최대 **0.635** |
  | 동일 인물 파편(아동 교차연령) | 최소 0.597 |
  | 동일 인물 파편(성인 VGGFace2) | 최소 0.625, 중앙 0.787 |
  0.6은 성인 파편을 더 구제하지만 서로 다른 아동을 오병합한다(child 데이터셋에서 0.7=오병합 0건 →
  0.6=오병합 2건). 사진 공유 앱에서 서로 다른 사람을 한 앨범에 섞는 오병합은 앨범이 둘로 갈리는
  것보다 훨씬 나쁘다. **소규모 단일 인물 앨범 생성은 병합이 아니라 부분 승격이 담당**하므로(단일
  인물 소규모 이벤트는 대개 클러스터 0개 갈래 → 승격 경로), 병합은 보수적으로 두는 것이
  "오병합보다 미병합"(ADR 006) 원칙에 맞다.
- **선택 — 연결 성분 + 완전 연결 floor + peel**: 간선(0.45)은 실측 동일 인물 하한(0.46) 직하에서
  소속을 판정하고, floor(0.4)는 성분 내부의 완전 연결을 보증해 체이닝 오병합을 차단한다. peel은
  극단적 포즈 한 장이 성분 전체를 무너뜨리는 것을 막는다. **승격되는 클러스터는 항상 완전 연결
  ≥ floor를 만족하므로(peel 불변식), 타인 쌍(<floor)이 같은 앨범에 남는 일은 절대 없다** — peel은
  기각 대신 최대 완전 연결 부분집합을 salvage할 뿐 안전 보장을 약화하지 않는다.

**기본값 근거** (실측 분포: 동일 인물 쌍 0.46~0.70, 타인 ≲0.3):

| 필드 | 기본값 | 근거 |
|------|--------|------|
| `blob_promote_similarity` | 0.45 | 동일 인물 하한 0.46 직하 (실사진 포섭), 타인 상한 대비 +0.15 마진 |
| `blob_promote_floor` | 0.4 | 타인 상한 +0.1 (체이닝 차단). `min_membership_similarity`(0.4)와 같아 승격 멤버의 LOO 유사도 ≥ floor가 보장되어(‖타멤버 합‖ ≤ n−1) 같은 실행의 저신뢰 축출 재강등이 없다 |

**다인물 순도가 깨지지 않는 이유**: 승격·peel은 전원 노이즈(클러스터 0개)일 때만 동작한다 —
HDBSCAN이 클러스터를 하나라도 만든 보통의 다인물 이벤트에는 무접촉이다. 병합 임계는 보수적
0.7 유지. 승격 뒤에도 cannot-link 강제·저신뢰 축출이 기존 순서대로 적용된다.

## Consequences

- 소규모 단일 인물 이벤트에서 인물 앨범이 생성된다 (face-test/images 실측: embedding 6장→5장 한
  앨범, karina 5장→4장 한 앨범, child 4장→4장 한 앨범). 나머지 1~2장은 유사도가 낮은 극단 포즈로
  보류(uncertain)되며, 이는 오병합을 피한 정직한 결과다. 동일 인물 2장(쌍 ≥0.45)도 구제 범위.
- "단일 인물 + 행인" 클러스터 0개 이벤트에서 행인만 노이즈로 남는다 (일괄 승격으로는 불가능).
- 구 0.7 완전 연결 fast-path는 제거 — `gram.min() ≥ 0.7`이면 새 로직이 정확히 같은 결과를 내고
  (전체가 한 성분 + floor 통과), 쌍별 임계에 centroid 병합 임계를 재사용하던 결합이 끊어진다.
- **알려진 한계 1 (파편화 갈래)**: 동일 인물이 HDBSCAN에서 `[3, 2]`로 파편화되고 파편 centroid가
  [0.625, 0.7)이면 병합 0.7이 못 이어 앨범이 2개로 갈린다 (성인 단독 이벤트에서 관측). 오병합
  회피를 위한 의도된 트레이드오프이며, 사용자 병합 보정으로 교정 가능하다.
- **알려진 한계 2**: 교차 쌍이 전면 floor 이상인 진짜 닮은 두 사람(쌍둥이·유아)은 한 성분으로
  승격될 수 있다 — 유사도 기반 방법의 공통 한계이며 cannot-link 보정이 교정 수단이다.
- 회귀 고정: `cluster.py __main__` 합성 스위트 13건(실측 대역 5장·행인 혼입·두 인물 분리·근중복
  버스트·체이닝 남남 분리·peel 극단 포즈·불변식) + `handlers.py __main__` 종단 케이스 1건(단일
  인물 5장 classify → 앨범 1개·uncertain 0건). e2e 케이스는 HDBSCAN 클러스터 0개 전제부터 검증한다.
- 실사진 검증 (로컬 `scripts/classify_local.py` + face-test/images 7종 192장): 동일 인물 소규모
  3종 전부 앨범 생성, 다인물(2명·10명) 순도 100%, 아동 교차연령(8명) 오병합 0건.
