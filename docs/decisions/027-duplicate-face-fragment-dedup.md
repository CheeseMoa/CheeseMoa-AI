# ADR 027: 초대형 얼굴 파편 이중 검출 디둡 — 1인 사진의 공용 앨범 오노출 해소

- 상태: 채택 (2026-07-21)
- 관련: CHMO-384, ADR 017/023(파편 bbox·재검출), ADR 011(같은사진 cannot-link의 이중 검출 안전판),
  ADR 022/025(주 인물 머릿수 자격), CHMO-330(단체 사진 공용 노출)

## 문제

group 37 / event 105의 앨범 538에서 혼자 찍힌 셀피가 **공용 앨범에도 노출**됐다. 원인은 다른 물체의
오검출이 아니라 **같은 얼굴 하나에 파편 bbox 2개**가 잡힌 것 — YuNet이 학습 분포(WIDER FACE, 작은
얼굴) 밖의 초대형 얼굴을 파편 박스로 그리는 알려진 현상(ADR 017/023)인데, 이번엔 파편 2개가 **둘 다
score 게이트(0.6)를 통과**(0.769/0.638)해 정규(confident) 검출이 됐다. 방어막 3중이 전부 미발동:

| 방어막 | 값 | 결과 |
|---|---|---|
| YuNet 자체 NMS (0.3) | 파편 쌍 IoU **0.297** | 미발동 (아슬아슬 미달) |
| ADR 017 디둡 (`_duplicates_existing`) | — | **회복 경로 전용** — confident끼리는 실행 안 됨 |
| 이중 검출 안전판 (`_SAME_FACE_SIMILARITY` 0.95) | 쌍 유사도 0.979 | cannot-link 면제엔 적용 — **머릿수엔 미적용** |

그 결과 파편 2행은 안전판 덕에 같은 인물 앨범에는 얌전히 들어가면서, 라우팅 머릿수(CHMO-330 "주
인물 2명+ → 공용")는 행 수를 그대로 세어 1인 셀피를 단체 사진으로 오판했다.

## 실측 (2026-07-21)

### 검출 층 — 원본 1,078장 전수 재검출, confident 쌍 2,921개 기하

같은 이미지 confident 쌍의 "정제 랜드마크 중심거리 / min(얼굴폭)" 분포:

- **진짜 파편 쌍** (유니크 2장 — event 105 셀피, 옛날 인화 아이 PNG. 재업로드 미러 포함 15쌍, 육안
  확정): **0.019, 0.024**. 폭은 1201·1079 / 535·559 — 전부 정제 대상(>224px) 초대형 얼굴.
- **실제 타인 겹침 쌍 최저**: **0.436** (뒤 인물에 겹친 옆얼굴, 폭 180·162 — 육안 확정). 그다음
  0.679(포스터 얼굴들)·0.712(배경 인물)·0.817(볼 맞댄 두 아이) — 전부 실제 타인, 육안 확정.
- bbox IoU는 판별력 없음: 파편 0.17~0.30 vs 실제 타인 최대 0.157 — NMS 조정으로는 못 가른다.

**최초 제안값 0.5는 기각** — 실제 타인 쌍 0.436을 삼켜 진짜 얼굴을 지운다(실측이 잡은 회귀). 빈
구간 [0.024, 0.436]의 기하 중앙 **0.1** 채택 + 파편화가 관측된 도메인(양쪽 폭 > refine_norm 224px)
한정으로 위험했던 소형 실얼굴 쌍을 원리적으로 배제한다.

### 라우팅 층 — 24개 이벤트 .npz, 같은사진 쌍 758개 유사도

- **이중 검출 쌍 6개 전부 0.978~0.979** vs **같은사진 타인 쌍 최고 0.756** — 기존 안전판 값 0.95가
  빈 구간 한가운데 (ADR 011 시절 실측 "타인 최대 0.61"의 재확인·갱신).
- 머릿수 붕괴 적용 시 라우팅 변화 = **정확히 6장**(event 85·90·91·104 각 1 + 105 2, 전부 이중 검출
  육안/기하 확정), 전부 "공용 → 인물 앨범만". 신규 공용 진입 0, uncertain 전환 0.

## 결정 — 두 층 (각각 독립 토글)

**① 검출 층: confident 파편 디둡** (`detect._dedup_confident_fragments`,
`DETECT_CONFIDENT_DEDUP_LANDMARK_RATIO=0.1`, 0=비활성) — 재발 방지(신규 데이터).
정제 완료 후 confident 얼굴을 score 내림차순으로 훑어, 양쪽 폭 > `refine_norm_face_width` AND
(정제 랜드마크 중심거리 < 0.1×min폭 OR IoU ≥ 0.5)면 같은 얼굴의 파편으로 보고 최상 박스만 남긴다
(회복 경로 관례와 동일). 정제 뒤에 실행하는 이유: 파편들의 랜드마크는 정제(ADR 023)가 같은 진짜
얼굴로 교정해야 중심이 모인다.

**② 라우팅 층: 머릿수 이중 검출 붕괴** (`handlers._headcount_eligible`,
`CLUSTER_COMMON_DUPLICATE_FACE_SIMILARITY=0.95`, 0=비활성) — 저장 데이터 치유.
같은 사진의 근중복(≥0.95) 연결 그룹은 폭 최대 행만 머릿수 자격을 유지한다. ①만으로는 이미
`.npz`에 저장된 파편 행이 안 고쳐진다 — 재분류는 기존 행을 재검출하지 않는다.
부수 보강: ADR 025 실인물 자격의 최근접 비교 상대에서 **같은사진 근중복 행만** 뺀다 — 오검출이
이중 검출되면 파편끼리 ~0.98로 바닥(0.185)을 뚫는 구멍 봉쇄. 같은사진 **전면** 제외는 기각 —
그 사진에만 등장하는 낯선 단체(서로만 닮은 얼굴들)를 오검출로 오판한다(자가검증 img-group이 적발,
실 이벤트에선 diff 0이라 표본 밖 위험이었다).

의미 변화 하나: 미배정 근중복 쌍만 있는 사진(자가검증 img-twins)은 이제 "단체 → 공용"이 아니라
"1인 미매칭 → uncertain"이다 — 0.95 이상은 타인이 아니라는 안전판 의미와 정합.

## 검증

- 자가검증 `python -m app.handlers` **46건 통과** — 신규 ⑲(파편 2행 사진은 인물 앨범만 + 붕괴
  0(비활성)이면 공용 노출 재현하는 적대적 대조), ①·⑥은 img-twins 의미 변화 반영해 갱신.
  `python -m app.worker --smoke` 9건 통과.
- 검출 diff (원본 1,081장, 디둡 OFF/ON 나란히 실행): **진짜 파편 사진 15장(유니크 2 + 재업로드
  미러)에서만 저score 파편 1개 제거, 그 외 완전 무변화** — 실제 타인 겹침 쌍(0.436) 무사.
- 라우팅 diff (24개 이벤트, 실제 `_headcount_eligible` OFF/ON): **사전 실측과 동일한 6장만 공용에서
  빠짐**, 그 외 무변화.
- 도구: `scripts/survey_confident_dup.py`(기하 전수 측정 + 크롭 시트),
  `scripts/verify_dup_dedup.py`(검출 OFF/ON diff + 라우팅 OFF/ON diff).

## 한계

- 정제가 파편 한쪽에서 실패하면(재검출 미매칭) 중심이 안 모여 ①이 놓치고, 임베딩도 갈라져 ②도
  놓칠 수 있다 — 미관측(코퍼스 파편 전부 정제 성공), 놓치는 방향이라 실얼굴 피해는 없다.
- `.npz`의 파편 행 자체는 남는다(centroid·썸네일 대표 선정에 미세 영향) — 행 정리는 P0 재업로드
  중복 정리(백로그)와 같은 데이터 정리 계층의 몫.
- 기하 임계(0.1)의 근거 표본이 얇다 — 진짜 파편 유니크 2장, 위험 실쌍 유니크 1장(0.436). 리포트
  축적 시 재보정. ①의 생존자는 score 최상(폭 아님) — ②의 폭 최대와 기준이 다르나 관측 사례에선
  결과 동일, 파편 bbox는 어차피 전부 부분 박스다.
