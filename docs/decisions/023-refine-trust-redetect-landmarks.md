# ADR 023: 재검출 랜드마크 신뢰 임계 — 초대형 얼굴 파편 bbox의 이동량 가드 오판 해소

## Status

Accepted (2026-07-17). `detect.py`의 랜드마크 이동량 가드(`_REFINE_MAX_SHIFT_RATIO`)에 신뢰
예외를 추가 — 정규 스케일 재검출 score가 `refine_trust_redetect_score`(기본 0.80, ADR-017 회복
임계와 동일값) 이상이면 이동량과 무관하게 재검출 랜드마크를 채택한다. refine(확정 얼굴 정제)·
recover(저score 대형 회복) 두 경로 공통. `DETECT_REFINE_TRUST_REDETECT_SCORE`, 0 = 비활성(종전 가드만).

## Context

group 27 / event 73 실 사용자 리포트: 얼굴이 크게 나온 1인 아기 사진(`fb0e2b5c…`)이 공통
사진첩(분류가 어려워요)으로 빠졌다. 로컬 재현으로 단계를 추적한 결과:

1. 실제 얼굴 폭은 이미지 긴 변의 ~65%인데 YuNet은 파편 박스 4개(rel_w 0.34~0.36)만 냈고,
   최고 score가 **0.640** — score 게이트(0.6)를 살짝 넘겨 "정상 검출"로 통과했다. score<0.6만
   판정하는 ADR-017 회복 경로는 아예 타지 않는다.
2. 대형 얼굴 랜드마크 정제(`_refine_face`)의 정규 스케일 재검출은 score 0.860으로 올바른
   랜드마크를 찾았다. 그러나 이동량 252.7px가 가드 상한(0.5 × **파편 bbox 폭** 465px = 232.5px)을
   넘겨 교정이 폐기되고 깨진 원 랜드마크(볼·이마에 찍힘)가 유지됐다.
3. 어긋난 정렬로 만든 임베딩은 같은 아기의 다른 사진과도 유사도 0.24(이벤트 내 최고 0.253,
   정상 동일인 쌍은 0.55~0.88) — HDBSCAN 노이즈 → 구제 불가 → unmatched → 공통 사진첩.

가드의 기준이 원 bbox 폭인 것이 핵심 결함이다: 초대형 얼굴은 bbox 자체가 파편이라(ADR-017
§재보정과 같은 YuNet 습성) 진짜 얼굴이 파편보다 훨씬 크고, **올바른 교정일수록 이동량이 크다**.
"이동이 크면 오매칭"이라는 가드 가정이 이 구도에서 역전된다.

같은 뿌리의 두 번째 증상도 발견됐다: recover 경로는 이동량 가드에 걸리면 "얼굴은 살리되 원
랜드마크 유지"로 폴백하는데, 대형 아기 얼굴 3장(`d6c27bde…`·`8cc345ae…`·`8f641976…` 로컬 파일
기준)에서 위로 어긋난 offset 파편 박스(score 0.21~0.29)가 이 폴백으로 쓰레기 랜드마크를 유지한
채 되살아났다. 랜드마크 중심이 본 얼굴과 멀어 디둡(ADR-017)도 비껴가고, "같은 아기의 이마+눈
크롭" 임베딩 3개가 서로 뭉쳐(0.47~0.56) **유령 인물 앨범**(`741d1aef…`, 사진 3장)이 프로덕션에
실제로 생성됐다. 같은사진 cannot-link(ADR-011)가 본 얼굴 클러스터와의 병합을 능동적으로
차단하기까지 한다.

## 실측 — 판별축 (scripts/survey_refine_shift.py)

34개 이벤트 520장 전수: 재검출 시도 1,043건(성공 471), 가드 발동 33건(유니크 13). 발동 사례를
원/재검출 랜드마크 양쪽 정렬 crop + 이벤트 npz 최근접 유사도(NN)로 판별:

| 재검출 score | 건수 | NN 변화(원→재검출) | 판정 |
|---|---|---|---|
| ≥ 0.86 | 좋은 교정 전부 | 0.32→0.98, 0.53→0.97, 0.25→0.69 등 전부 개선 | 채택해야 함 |
| ≤ 0.39 | 나머지 전부 | 0.17→0.19 등 양쪽 다 쓰레기 | 어차피 회복 임계(0.8) 미달로 폐기 |

빈 구간 **[0.392, 0.860]** — ADR-017이 이미 검증한 회복 임계 0.80이 정확히 이 안에 있어 같은
값을 채택한다("정규 스케일 재검출 score ≥ 0.80 = 실얼굴"이라는 기존 판별 기준의 재사용).
크롭 육안 검증에서 신뢰 채택 대상 전건이 같은 얼굴의 중앙 정렬 교정이었고, 옆 사람 얼굴로
점프한 오매칭은 0건. shift/재검출폭 축은 판별력 없음(좋은 교정도 0.83까지 올라가 통과 상한
0.617과 겹침) — score가 유일하게 깨끗한 축이다.

## Decision

`DetectorConfig.refine_trust_redetect_score = 0.80` (기본값): `_normal_scale_redetect`가 성공하고
재검출 score가 이 값 이상이면 이동량 가드를 무시하고 재검출 랜드마크를 채택한다.

- `_refine_face`: 종전엔 이동량 초과 시 무조건 원 랜드마크 유지 → 이제 신뢰 임계 이상이면 채택.
- `_recover_large_face`: 종전엔 이동량 초과 시 원 랜드마크로 회복(쓰레기 임베딩 생성) → 이제
  신뢰 임계 이상이면 재검출 랜드마크로 회복. 교정된 랜드마크 중심은 본 얼굴을 가리키므로
  offset 파편 박스는 디둡(`_duplicates_existing`)에 걸려 제거된다 — 유령 앨범의 뿌리 차단.
  기본값에서는 회복 임계(0.80)와 신뢰 임계(0.80)가 같아 "원 랜드마크 유지" 폴백이 사실상
  죽은 가지가 되는데, 이것이 의도다(신뢰 임계를 0.80보다 올리면 폴백이 되살아난다).

이동량 가드 자체는 저score(<0.80) 재검출에 대해 그대로 남는다. 오매칭 1차 방어는 종전처럼
`_REFINE_MATCH_IOU`(0.3)가 담당한다.

## 검증

- **34개 이벤트 전 경로 diff** (old=비활성 vs new, JobHandlers classify 동일 경로): 31개 완전
  불변. 변경 3개(66·68·73 — 같은 사진 세트)는 전부 의도된 수정: 리포트 사진이 uncertain에서
  빠져 본인 앨범(11→12장) 편입 + 유령 앨범(3장) 소멸. 의도 밖 변화 0.
- **라벨 코퍼스**: different·similar·similar2·similar3 얼굴 수·파티션 완전 동일. child는 얼굴
  60→54 — 사라진 6개 전부 score 0.21~0.42의 offset 파편 박스(디둡 정상 동작, 실얼굴 손실 0),
  ARI **0.573→0.794**(동일 측정 방식 비교), 노이즈 10→6. 회귀가 아니라 순개선.
- 워커 스모크(`--smoke`) 8건 전부 통과.

## 한계

- 신뢰 채택의 오매칭 안전성은 코퍼스 실측(발동 33건 중 점프 0)에 근거한다. 대형 얼굴 두 개가
  겹치듯 밀집한 구도(IoU 게이트 0.3을 옆 얼굴이 통과할 만큼 근접)는 코퍼스에 표본이 없다 —
  오분류 리포트 축적 시 `_REFINE_MATCH_IOU` 상향과 함께 재실측.
- score 게이트(0.6)를 넘긴 파편 bbox는 여전히 bbox 자체는 파편으로 남는다(정제는 랜드마크만
  교체, bbox는 하류 계약상 유지). 품질 게이트의 rel_w 판정(주 인물 비율·붕괴 면제)이 파편 폭을
  기준으로 동작하는 것은 별도 문제로 남는다 — 이번 코퍼스에서는 증상 없음.
