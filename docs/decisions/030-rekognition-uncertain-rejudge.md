# ADR 030: Rekognition uncertain 재판정 — 하드케이스 자동 편입·제안

- 상태: 채택 (2026-07-24) — 구현 완료, **기본 활성화(CHMO-420, `REJUDGE_ENABLED=true` 기본)**.

- 관련: [2026-07-23 실측 리뷰](../reviews/2026-07-23-rekognition-uncertain-ab.md)(근거 데이터 전체),
  [파편병합 완화 기각](../reviews/2026-07-23-merge-facepair-relaxation-rejected.md)·
  [가림·자세 갈라짐 실측](../reviews/2026-07-23-occlusion-pose-split-survey.md)(로컬 임계 불가 결론),

## 문제

옆얼굴·가림·나이차 하드케이스는 AuraFace 코사인 유사도로 동일인과 타인을 원리적으로 못 가른다
(동일인 하드 0.18~0.48 vs 타인 0.20~0.36 — 완전 겹침). 어떤 로컬 임계 조정으로도 회수 불가가
같은 날 두 리뷰로 확정됐고, 모델 교체는 라이선스로 봉쇄됐다(2026-07-14 조사). 그 결과 실서버
미배정 얼굴의 상당수(측정 가능분 392개)가 "분류가 어려워요"에 영구히 남는다.

AWS Rekognition CompareFaces는 같은 쌍을 동일인 83~100 vs 타인 0~16으로 갈랐고(남매만 57),
실서버 전수 스윕에서 임계 95 기준 96개(24.5%)를 육안 전수 정답으로 회수했다. 관리형 API는
라이선스 봉쇄 밖이고, 이미 수탁자인 AWS 서울 리전이라 신규 제3자 제공·국외 이전이 없다.

## 결정

재군집이 끝나고 uncertain으로 확정되기 직전, 핸들러가 미배정(노이즈+ambiguous) 얼굴을 **AuraFace
유사도 상위 `top_k`(3)개 앨범의 대표 얼굴과 CompareFaces로 비교**해 최고점으로 재판정한다
(`app/pipeline/rejudge.py`의 `UncertainRejudger`, 배선은 `core/deps._build_rejudger`):

- 최고점 **≥ 90**(`REJUDGE_AUTO_ASSIGN_SIMILARITY`, 보수 95) → 그 앨범에 **자동 편입** —
  `(얼굴, 대표)` 쌍을 `must_link_pairs`에 기록하고 재군집을 한 번 더 돌린다(§2차 패스).
- 최고점 **[85, 90)**(`REJUDGE_SUGGEST_SIMILARITY`) → 결과 메시지 `uncertain[].suggestions`에
  **"이 앨범 아닐까요?" 제안** 동봉(계약 확장 — feature-spec §6.2·message-examples §④, Spring 합의
  필요. 수락은 기존 `__uncertain__` reassign 재사용). 85 미만은 오답 대역(82~84 남매·타인 아기
  실존)이라 제안조차 하지 않는다.
- **≥ 95 복수 앨범 동시 매칭**(`REJUDGE_FRAGMENT_SIMILARITY`) → 같은 인물의 파편 앨범 힌트로
  INFO 로그 기록 (wire 계약은 후속 — 실측 20건 대부분 파편).
- **조기 종료 없음** — top_k 전 후보를 호출한 뒤 argmax로 판정한다. 이 기능이 겨냥하는 대역이
  바로 AuraFace 순위를 못 믿는 대역이라, 1순위가 임계를 넘겼다고 멈추면 2순위가 진짜 정답일 때
  오답 앨범에 붙는다. 아끼는 비용은 얼굴당 최대 $0.002로 실익이 없다.
- 한 얼굴의 top-k 쌍 점수가 **전부 확보돼야 판정**한다(호출 상한·crop 실패·일시 장애로 일부만
  확보되면 그 얼굴은 판정 보류) — 부분 점수의 argmax는 미측정 앨범이 정답일 때 오답 편입이 된다.

### 자동 편입 = must-link 기록 + 재군집 2차 패스 (스냅샷 직접 수정 기각)

편입을 재군집 결과에 반영하는 방식으로 **"must-link 추가 후 `_cluster_pass` 재실행(1회)"**을
택했다. 스냅샷(멤버 목록) 직접 수정은 같은사진 자동 cannot-link·단일 사진 강등·파편 병합 게이트·
저신뢰 축출 등 모든 후처리 불변식을 손으로 재검사해야 하고, 무엇보다 **저장된 `.npz`를 결정적
재군집한 결과와 발행 결과가 어긋나** 재전달(at-least-once) 안전성이 깨진다. 2차 패스는 모든
불변식을 자동 재적용하며 비용은 전체의 0.1% 미만이다. must-link 당사자는 축출·강등에서 보호되므로
편입이 후처리에 뒤집히지 않고, 저장된 must-link 덕에 **이후 어떤 재군집에서도 그 얼굴은 uncertain에
재진입하지 않는다**(재호출 루프 원천 차단). 2차 패스가 새 uncertain을 만들어도 루프 없이 1회로
끝낸다 — 캐시 덕에 다음 job에서 이어 판정한다. **event 교체는 2차 패스 성공 후에만 커밋**한다:
예상 밖 모순 제약이 저장되면 이후 모든 재군집이 죽는 오염을 막는다.

### 편입 전 3중 사전 검사 (기각은 제안 강등 없이 탈락)

1. **사용자 cannot-link 존중**: 얼굴(의 must-link 컴포넌트)이 대상 앨범 멤버(의 컴포넌트)와
   cannot-link로 묶여 있으면 점수와 무관하게 기각 — 사용자가 "다른 사람"이라 확정한 결정이
   기계 증거에 밀리면 안 된다. 제안도 같은 검사를 통과해야 한다.
2. **같은 사진 공존 검사**: 대상 앨범에 같은 `photo_id`의 (파편이 아닌, 유사도 < 0.95) 얼굴이
   있으면 기각 — 같은 사진에 같은 인물이 두 번 나올 수 없고, 편입하면 2차 패스의 같은사진 자동
   cannot-link가 앨범을 쪼갠다.
3. **사진·앨범당 argmax 가드**: 같은 사진의 두 얼굴이 같은 앨범에 동시 편입되면 두 must-link가
   한 컴포넌트로 합쳐지고 그 사이 자동 cannot-link는 사람 결정 우선 규칙으로 탈락해 서로 다른
   두 사람이 조용히 융합된다 — 최고점만 남긴다.

### 점수 캐시 — event별 S3 JSON (`rekognition-scores/{event_id}.json`)

`(source_face_id, rep_face_id) → Similarity` 맵을 워커 소유 `embeddings_bucket`에 보관한다
(`app/storage/rekognition_scores.py`). **재군집·보정마다 재판정이 다시 돌아도 같은 쌍은 캐시
히트라 재과금이 없다** — 전체 재군집 정책(ADR 007)과 호출당 과금 API의 양립 장치. face_id 키라
행 인덱스 변동에 안전하고, "crop에서 얼굴 미검출"은 `-1.0` 센티널로 캐싱해(영구 성질) 일시
장애(스로틀 — 미캐싱, 다음 job 재시도)와 구분한다. 저장 시 죽은 face_id 항목은 프루닝하고,
event 전체 삭제 시 캐시 객체도 삭제한다(생체 파생 정보가 원본보다 오래 남지 않게). npz 컬럼
추가(v5)는 기각 — 가변 길이 희소 쌍 맵이 고정 행 레이아웃에 맞지 않고 매 재군집 전체 rewrite를
불린다. 손상 캐시는 로그 후 빈 캐시 취급(소모품 — `.npz`의 보존 정책과 의도적으로 다름).

### 앨범 대표·crop — 실측 파리티가 계약

- 대표 = 멤버 중 **폭 최대 얼굴**(bbox·s3_key 유효 행만). 썸네일 대표(LOO membership 최고)와
  의도적으로 다르다 — 임계 90/85/95의 근거 데이터가 폭 최대 대표로 측정됐다.
- crop = **bbox + 0.25 여백, JPEG q92, 축소 없음**(`render_rejudge_crop`) — 근거 데이터와 동일
  파라미터. 썸네일 렌더러(1.4배 확장 + max_side 축소)를 쓰면 근거가 무효가 된다.
- 원본은 행 단위로 1장씩 fetch·폐기하고 crop bytes만 job 내 캐시한다(t4g.small 메모리 규율).

### 안전판·격리

- **best-effort** (썸네일과 동일 정책): 재판정·캐시의 어떤 실패도 job을 죽이지 않는다 — 실패 시
  1차 패스 결과 그대로(현 동작 유지)이며, 전면 장애 시 결과가 비활성 실행과 동일함을 자가검증이
  고정한다.
- **호출 상한** `REJUDGE_MAX_CALLS_PER_JOB`(150) — 오염 이벤트의 비용 폭주 안전판. 초과분은
  판정 보류 + `truncated` 로그(침묵 절단 금지).
- 제안의 앨범 참조는 판정 시점 cluster_id가 아니라 **대표 얼굴이 최종 저장 상태에서 속한
  클러스터**로 재해석한다 — 최초 분류에서 1차 패스의 신규 id가 2차 패스에서 재발급되는 문제,
  병합·은퇴로 id가 바뀌는 문제에 모두 강건하다.

## 활성화 게이트 (`REJUDGE_ENABLED=false`가 롤백 스위치)

기본값은 CHMO-420에서 ON으로 전환됐다. 실사용자 사진을 처리하는 운영 환경의 전제 조건
(리뷰 §도입 선행 조건, CLAUDE.md 목표 0) — 미충족이면 해당 환경은 false로 배포한다:

1. **AWS AI 학습 opt-out** — 현 Innovation Sandbox 계정은 Organizations API 차단으로 설정 불가
   실측(2026-07-23). 샌드박스 운영자에게 정책 부착 요청 또는 정식 계정 이전이 선행.
2. 개인정보처리방침 위탁 항목에 얼굴 분석 업무 명시 (PIPA 제26조).
3. "얼굴인식 특징정보(민감정보) 처리" 동의 문구 점검 — 생체정보+아동 조합은 전문가 검토 권장.
4. 워커 IAM에 `rekognition:CompareFaces` 권한 추가 (SCP는 서울 리전 허용 실측).

## 검증

- `python -m app.pipeline.rejudge` — 판정 로직 24건 (편입/제안 대역/3중 검사/상한 절단/캐시
  히트 0호출/-1.0 센티널/crop 파리티).
- `python -m app.storage.rekognition_scores` — 캐시 코덱 왕복·손상·멱등 삭제.
- `python -m app.handlers` — 종단 10건: 편입→npz must-link→재분류 시 uncertain 재진입 없음·추가
  호출 0회, 제안 face_bbox 결속, cannot-link 존중, 전면 장애 = 비활성 동일 결과, 전체 삭제 시
  캐시 삭제, 웜 캐시 콜드 시작 0호출.
- `python -m app.worker --smoke` — 발행 경유: 편입 얼굴이 clusters에, 제안이 suggestions에 실려
  JSON 왕복.
- 합성 픽스처 주의: 동일인 벡터 쌍은 근중복 붕괴 임계(0.985, ADR 029) **아래**로 만들 것 — 위면
  ⓪ 붕괴+고아 승격 경로에 얹혀, 편입 must-link가 2차 패스에서 클러스터를 만들 때 승격 조건이
  꺼져 무관한 앨범이 와해된다 (handlers ㉓ 픽스처 주석).

## 한계

- Rekognition 오류 대역이 실존한다: 남매 57점, 저화질 형제/쌍둥이 아기 94~98점대 — 자동 편입
  임계(90)가 방어하지만, 아기 다수 이벤트는 보수 임계(95) 또는 제안 전용 운영 권장.
- 커버리지는 원본이 남은 얼굴뿐(bbox·s3_key 유효). 구버전 `.npz`(v2 이하) 행과 원본 소실분은
  재판정 불가 — S3 버저닝(backlog P0 §1)의 필요를 재확인.
- 파편 병합 힌트는 로그 전용 — 자동 병합·wire 계약은 후속 결정.
- 제안 UX는 Spring·앱 구현 전까지 무시되는 필드다(빈 배열 하위 호환).
