# ADR 019: 눈감음 판정 자격 게이트 — 최소 얼굴 크기 + 눈/볼 밝기 비 상한

## Status

Accepted (2026-07-17). 눈감음 품질 게이트(CHMO-172)의 판정 전 자격 검사 추가 —
`judge_faces`의 눈 상태 CNN 호출 앞단에 적용. "눈이 안 보이는(신뢰 불가한) 얼굴은 눈감음
판정 대상이 아니다"를 계약으로 만든다.

## Context

웃음 캔디드 오탐 조사([2026-07-17 survey](../reviews/2026-07-17-smile-eyes-geometry-survey.md))가
드러낸 실측 사실: 실 코퍼스 859 얼굴에서 현행 eyes_closed 판정(양눈 closed prob ≥ 0.85)에 걸리는
11건(고유 8건) 중 정탐은 1건뿐이고, 오탐 유형은 웃음이 아니라 **① 초소형 그림·포스터 얼굴/옆얼굴
(32~52px) 4건, ② 고글·마스크 착용 얼굴(눈 뜸) 2건, ③ 눈 뜬 아기 1건**이었다. 선글라스·고글
얼굴은 눈 crop에 눈이 없어 CNN 출력 자체가 무의미한데도 높은 확신으로 closed를 내고, 해당
사진은 인물 앨범 대신 eyes_closed 앨범으로 빠진다.

### 판별 신호 실측 (859 얼굴, `scripts/survey_eye_occlusion.py`)

가림 검출용으로 가정한 두 신호는 **기각**됐다:
- 눈 영역 어두움(눈/볼 밝기 비 하한): 정상 눈도 눈동자·속눈썹으로 어두워(중앙값 0.80) 짙은
  렌즈(0.54~0.61)와 겹친다. 느슨한 후보 바운드(≤0.75)에 전체의 34%가 걸림 — 판별축 아님.
- 눈 박스 매끈함(low variance): 선글라스 렌즈는 반사·프레임 때문에 오히려 텍스처가 매우 높다
  (3291~4933 vs 전체 중앙값 1482) — 가설과 반대 방향.

대신 오탐 코호트가 두 축에서 갈렸다:

| 게이트 | 오탐 위치 | 정상 분포 | 면제 비용 (859 얼굴) |
|---|---|---|---|
| bbox 짧은 변 < 64px | 그림·옆얼굴 오탐 4건 전부 32~52px | 정탐(기도) 643px | 81 얼굴(9.4%) 면제, 그중 진짜 눈감음 0 |
| 눈/볼 밝기 비 > 1.4 | 고글+마스크 2건이 1.73·2.75 | p95 1.21, 정탐 0.79 — 빈 구간 [1.21, 1.73] | 26 얼굴(3.0%) 면제, 그중 CNN이 감음이라 한 것은 그 오탐 2건뿐 |

밝기 비 상한의 물리적 근거: 판정하려는 대상(감은 눈꺼풀)은 피부라서 볼과 밝기가 비슷해야 한다.
눈 박스가 볼보다 뚜렷이 밝으면 눈 자리에 피부가 아닌 것(고글 반사)이 있거나 볼 참조가 살이
아니라는(마스크) 이상 신호다. 최소 크기의 근거는 blur의 `min_blur_face_px`와 동일 — 눈 박스의
원본 정보가 몇 px뿐이면 CNN 입력이 업샘플 뭉개짐이라 신뢰 불가.

## Decision

`judge_faces`의 눈감음 판정에 자격 게이트 2개를 추가한다 (`_eye_judgment_eligible`):

1. **최소 얼굴 크기** `min_eye_face_px`(기본 64): bbox 짧은 변이 미만이면 미판정.
2. **눈/볼 밝기 비 상한** `eye_cheek_brightness_ceiling`(기본 1.4): 정렬 crop의 고정 눈 좌표
   박스 평균 밝기 / 같은 x의 볼(눈 아래 26px) 평균 밝기의 양눈 평균(`eye_cheek_ratio`)이
   초과면 미판정. 측정 불가(박스가 경계에 잘림)도 보수적으로 미판정.

자격 미달은 "옆얼굴 한쪽 눈 미검출"과 같은 기존 보수 정책의 확장이다 — 눈 상태를 모르는 것이지
눈을 감은 것이 아니므로, 그 얼굴은 건너뛰고 다른 얼굴로 판정을 계속한다. 둘 다
`QualityConfig`/`Settings`(.env `QUALITY_MIN_EYE_FACE_PX`, `QUALITY_EYE_CHEEK_BRIGHTNESS_CEILING`)
설정값이며 0 = 비활성.

## Consequences

- 실측: 코퍼스 eyes_closed 이미지 11 → 5장. 해제된 6장 전부 오탐(포스터 4, 고글마스크 2),
  정탐(기도) 유지, 새로 생긴 eyes_closed 0 (`judge_faces` 전수 재실행으로 회귀 확인).
- 비용: 눈 상태를 판정하지 않는 얼굴이 늘어난다(크기 9.4% + 밝기 3.0%). 실측에서 이 면제군에
  진짜 눈감음은 0이지만, 코퍼스의 눈감음 표본 자체가 1건이라 미탐 리스크의 통계적 확증은 아니다
  — 실서비스 미탐 리포트가 쌓이면 임계 재보정.
- **남은 한계 ①**: 짙은 선글라스는 어두워지는 방향이라 밝기 비 상한으로 못 잡는다(하한은 정상
  눈과 겹쳐 기각). 코퍼스의 선글라스 오탐은 전부 초소형 그림 얼굴이라 크기 게이트에 걸렸지만,
  크게 찍힌 실물 선글라스 얼굴은 재현 표본이 없어 미검증 — 리포트 축적 시 별도 신호 필요.
- **남은 한계 ②**: 눈 뜬 아기 오탐 1건(522px, 밝기 비 0.55 정상 범위)은 게이트로 못 잡는다.
  open-closed-eye-0001이 운전자 모니터링 도메인 학습이라 유아 눈(큰 홍채 비율)에서 오판하는
  것으로 추정 — CNN 도메인 문제라 신호 게이트가 아닌 모델/임계 차원의 후속 과제.
- 웃음 캔디드 오탐(기존 기록 한계)은 실측 0건 — 표정 CNN 후속은 착수 근거 없음(survey 문서).
