# ADR 026: 눈감음 상대 크기 게이트 — 원거리 "아래 쳐다봄" 오탐 제외

## Status

Accepted (2026-07-21). `judge_faces`의 눈감음 판정(blendshape·CNN 롤백 경로 공통) 자격에
`eye_main_face_ratio`(최대 얼굴 대비)와 나란히 **이미지 대비** 크기 하한 `eye_min_rel_width`를
추가 — "프레임에서 작게 찍힌 원거리 얼굴의 눈 상태는 그 사진을 눈감음첩으로 보낼 근거가
아니다"를 계약으로 만든다. [ADR 022](022-eye-main-face-ratio.md)의 주 피사체 원리를 이미지
denominator로 확장한 것이다.

## Context

group 35 / event 99 실 사용자 리포트: 수박옷 아이가 눈을 감은 게 아니라 컵·식물을
**내려다보는데** eyes_closed로 분류돼 인물 앨범에서 빠졌다. blink blendshape(ADR 021)는
"윗눈꺼풀이 내려온 정도"를 재는데, 시선을 아래로 내리면 반쯤 감을 때와 같은 만큼 눈꺼풀이
내려온다 — "내려뜬 뜬 눈"과 "감은 눈"이 겹친다.

실측(전 이벤트 961장, 프로덕션 경로 동일 — 상세 [review](../reviews/2026-07-21-blink-lookdown-relwidth.md)):

**① threshold로는 못 가른다.** 오탐(내려뜸)의 blink가 정탐(감음) 분포 한가운데 박혀 있다:

| 얼굴 | blink | 실제 |
|---|---|---|
| 22a3d7f8 함성 질끈 | 0.516 | 감음 (정탐) |
| 66f8c71e 식물 내려봄 | 0.530 | 내려뜸 (**오탐**) |
| event 61 진짜 감음 | 0.625 | 감음 (정탐) |
| 04bebbd5 교실 뒤 | 0.625 | 내려뜸 (**오탐**) |
| daf7f144 먹으며 감음 | 0.637 | 감음 (정탐) |

event 61 정탐과 event 99 오탐이 소수점 셋째 자리까지 같은 **0.625** — 어떤 임계도 둘을 못 가른다.
blendshape eyeLookDown 축도 진짜 감음이 함께 떠서 기각.

**② 절대 px로도 못 가른다.** 정탐(함성 182px)이 오탐(수박옷 58px, 식물 188px) **사이에 끼어**
중첩. 짧은 변 64px 게이트(ADR 019)는 46·58px만 잡고 식물 188·210px는 못 잡는다(4개 중 2개).

**③ 상대 크기(rel_w = bbox 폭/이미지 긴 변)로는 갈린다.** 식물 사진은 3024×4032 고해상도라
아이 얼굴이 절대 188~210px여도 프레임에선 **4.7~5.2%**(멀리 찍힌 작은 피사체). 함성·먹는 감은
얼굴은 **14~17%**(가까운 주 피사체). 내려뜸 오탐은 멀리 찍힌 작은 얼굴, 진짜 감음은 가까운 주
피사체 — 이 상관이 판별축이 된다. 전 이벤트 flagged 얼굴 76개 분포:

| rel% | 대표 | 분류 |
|---|---|---|
| 2.5 ~ 5.2% | 식물 188·210px, 수박옷 58px, 교실 46px, 소형 21~32px | **내려뜸/소형 오탐** |
| 7.6 ~ 10.8% | 선글라스 434px, 손 가림 93px, 안경+모션 115px | 가림/모션 오탐 (별개 유형) |
| 11.3 ~ 37.8% | 기도 643px, 수면 538px, 함성 182px, 먹음 569px, 정면 감음 | **정탐** |

내려뜸 오탐은 전부 ≤5.2%, 진짜 감음은 전부 ≥11.3% — 그 사이 빈 구간.

이는 검출·threshold가 아니라 **판정 대상**의 문제다(ADR 022와 같은 성격): 눈감음 앨범의 목적은
"주인공이 눈 감아 버릴 사진"의 분리이지, 프레임에 작게 걸린 얼굴의 눈 상태 감사가 아니다.
`eye_main_face_ratio`(최대 얼굴 대비 0.5)는 식물 사진의 아이가 사진 내 최대 얼굴(솔로)이라
못 걸렀다 — denominator가 이미지여야 "프레임에서 작음"을 잡는다.

## Decision

`QualityConfig.eye_min_rel_width = 0.08` (기본값, `QUALITY_EYE_MIN_REL_WIDTH`, 0 = 비활성):
bbox 폭이 이미지 긴 변의 이 비율 미만인 얼굴은 눈감음 판정에서 제외한다(미판정). blink·CNN
롤백 경로 공통이며, `eye_main`과 AND로 `judge_faces` 눈감음 분기 앞단에 적용한다.
`judge_faces`에 `image_long_side` 파라미터를 추가해 이미지 긴 변을 전달한다(0 = 미상 → 게이트
무시, 안전 기본값).

**임계 0.08 선정**: 빈 구간 [5.2%, 11.3%]에서 채택 — 내려뜸 오탐 상한(5.2%) 대비 1.54배,
진짜 감음 하한(11.3%) 대비 1.41배 마진. 검출 rel 하한(`DETECT_MIN_FACE_REL_WIDTH` 2.5%,
ADR 013)은 앨범 편입용이라 더 낮다 — 얼굴은 앨범에 넣되 눈감음만 더 엄격히 요구한다.

게이트는 **단조적**이다(작은 얼굴의 eyes_closed 플래그를 제거만 할 뿐 새로 붙이지 않음).

## Consequences

**검증** (전 이벤트 959장, `eye_min_rel_width` 0 vs 0.08 이미지 단위 diff):
- 판정 변경 **37장, 전부 True→False(해제), False→True 0**.
- 해제 37장 = 내려뜸/소형 오탐 4종 × 미러 이벤트 8개(32) + 소형 얼굴(event 5·13·89, 4) +
  선글라스 가림 오탐(event 83, 1). **진짜 감음 이미지(≥11.3%)는 한 장도 안 뒤집힘 — 미탐 0.**
- event 99 국소: 오탐 4장(46·58·188·210px) 전부 해제, 정탐 2장(함성 182px·먹음 569px) 유지.
- 스모크 9건 통과.

**한계**:
- **프레임에서 큰 내려뜸**은 못 거른다 — event 103 아이(웃으며 내려봄, rel 32.8%)는 게이트
  위라 잔존. 크기 축의 원리적 사각지대(웃음 실눈 캔디드와 같은 의미론 문제).
- **가림/모션 오탐**(선글라스·안경)은 이 게이트 대상 아님 — ADR 019 계열의 별도 문제. 8% 아래로
  들어온 선글라스 1건(event 83)은 부수적으로 해제되나 의도된 타깃은 아니다.
- **원거리 진짜 감음**은 미판정 — 주 피사체가 아니면 눈감음첩 분리 대상이 아니라는 ADR 013·022
  원리상 허용되는 트레이드오프(사용자 확인). 리포트 축적 시 하한 재보정.

**롤백**: `QUALITY_EYE_MIN_REL_WIDTH=0`으로 게이트만 끈다(종전 동작). 도구:
`scripts/survey_eye_rel_width.py`(전 이벤트 flagged 얼굴 rel% 수집·크롭).
