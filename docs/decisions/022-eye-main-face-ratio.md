# ADR 022: 눈감음 주 인물 규칙 — 배경 얼굴 눈감음 판정 제외

## Status

Accepted (2026-07-17). `judge_faces`의 눈감음 판정(blendshape·CNN 롤백 경로 공통) 앞단에
blurry와 동일한 주 인물 자격 규칙을 추가 — "배경 행인이 눈을 감은 것은 그 사진을 눈감음첩으로
보낼 이유가 아니다"를 계약으로 만든다.

## Context

group 27 / event 69 실 사용자 리포트: 주 인물 2명이 모두 눈을 뜨고 선명하게 나온 사진
(`4f108f99…`, 4032×3024)이 eyes_closed 앨범으로 분류됐다. 로컬 재현(프로덕션 경로 동일) 결과:

| 얼굴 | bbox 폭 | 최대 얼굴 대비 | min(blinkL, blinkR) | 판정 |
|---|---|---|---|---|
| 주 인물 1 | 598px | 0.91× | 0.024 | 뜸 |
| 주 인물 2 | 656px | 1.00× | 0.017 | 뜸 |
| 배경 행인 | 101px | **0.15×** | **0.426** | 감음 (임계 0.40) |

blendshape 판정 자체는 틀리지 않았다 — 프레임 오른쪽 끝(x=3792/4032)에 걸린 군중 속 행인이
실제로 눈을 내리깔고 있다. 문제는 판정의 **대상**이다:

1. 이 얼굴은 rel_w 101/4032 = 0.02505로 배경 인물 크기 필터(ADR 013, 2.5% **미만** 제거)를
   0.00005 차이로 통과했다. ADR 013 하한 상향은 앨범 얼굴 최소값(3.29%)과 마진이 1.3배뿐이라
   위험하다 — 검출 단계에서 자를 수 없다.
2. 흔들림 판정은 같은 문제를 이미 겪고 해결했다(event 30, `blur_main_face_ratio` 0.5 —
   배경 인물의 아웃포커스는 흔들림 증거가 아님). 그러나 눈감음은 검출된 **모든** 얼굴을
   판정했고, "얼굴 1개라도" 이미지 집계 규칙이 행인 1명을 사진 전체의 eyes_closed로 승격시켰다.
   종전 CNN 경로의 ADR 019 자격 게이트(짧은 변 ≥64px)도 101×128 얼굴이라 못 걸렀을 크기고,
   blendshape 경로는 presence 게이트뿐인데 이 얼굴은 presence 1.0이었다.

사용자가 사진을 보고 "둘 다 눈 떴는데?"라고 느낀 것 자체가 이 기능의 사용자 의미론을 증명한다 —
눈감음 앨범의 목적은 "주인공이 눈 감아서 버릴 사진"의 분리이지, 프레임에 우연히 걸린 타인의
눈 상태 감사가 아니다.

## Decision

`QualityConfig.eye_main_face_ratio = 0.5` (기본값, `QUALITY_EYE_MAIN_FACE_RATIO`, 0 = 비활성):
사진에서 가장 큰 얼굴 bbox 폭 대비 이 비율 미만인 얼굴은 눈감음 판정에서 제외한다(미판정).
`blur_main_face_ratio`와 같은 값·같은 논리이며, blink(blendshape) 경로와 CNN 롤백 경로에
공통 적용된다. 별도 설정으로 둔 것은 독립 롤백·튜닝을 위해서다(blur 쪽 회귀 없이 눈감음만
끄거나 조일 수 있어야 한다).

## Consequences

- 검증 (2026-07-17, 로컬 라벨 코퍼스 155장 + event 69 원본 16장 = 171장 전/후 비교):
  - eyes_closed 뒤집힘은 정확히 그 행인 사진뿐(코퍼스 내 동일 사진 사본 포함 2건) — 오탐 해소.
  - 기존 eyes_closed 정탐 13장 전부 유지(감음 라벨셋 포함) — 무회귀. 감음 라벨셋의 감은
    얼굴은 전부 주 인물(최대 얼굴 대비 ≥0.5)이라 게이트에 걸리지 않는다.
  - `python -m app.worker --smoke` 8건 통과.
- 한계: **주 인물 2명 이상이 비슷한 크기로 나온 단체 사진에서 한 명이 감은 경우는 계속
  잡는다**(0.5 비율은 그런 얼굴을 배제하지 않음 — 의도된 동작). 반대로 주 인물이 정말로
  최대 얼굴의 절반 미만 크기인 구도(대형 얼굴 1 + 멀리 선 일행)에서는 작은 쪽 일행의 눈감음을
  놓친다 — blur 쪽과 동일한 트레이드오프이며, 미탐 리포트가 쌓이면 비율을 blur와 분리해
  재보정한다.
- 진단 CLI(`python -m app.pipeline.quality <이미지>`)가 배경 얼굴 제외를 표시한다
  (`배경 얼굴 — 눈감음 판정 제외`).
