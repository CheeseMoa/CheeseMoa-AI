# ADR 031: Rekognition 앨범 쌍 병합 재판정 — 동일 인물 앨범 쪼개짐 회수

- 상태: **제안 (2026-07-24) — 미구현.** 실측 근거는 확보됐고 설계는 아래대로 확정. 운영 전제 조건은
  ADR 030 §활성화 게이트와 동일(같은 API·같은 데이터 흐름이라 새로 생기는 항목 없음)

  [파편병합 완화 기각](../reviews/2026-07-23-merge-facepair-relaxation-rejected.md)(로컬로는 불가) ·
  [ADR 016](016-merge-facepair-cohesion-gate.md)(막고 있는 게이트) · [ADR 024](024-merge-component-linkage.md) ·
  [ADR 011](011-same-photo-cannot-link.md)(같은 사진 = 확정적 반증) · ADR 007(재군집 흐름·단일 writer)

## 문제

오병합은 거의 없는데 **같은 인물이 두 앨범으로 갈라지는 일이 반복**된다. 실측(2026-07-24)에서
앨범이 2개 이상인 34개 이벤트 중 **12개(35%)**에 회수 가능한 쪼개짐이 있었다.

주범은 파편 병합의 face-pair 응집 바닥(ADR 016, 0.475)이다 — 회수 대상 13쌍 중 **12쌍이 "centroid는
0.55를 통과했는데 face-pair가 0.475에 막힌"** 쌍이다. 그런데 이 바닥은 낮출 수 없다:
[기각 리뷰](../reviews/2026-07-23-merge-facepair-relaxation-rejected.md)가 실서버 전수로, 정당 병합의
최강쌍(0.524·0.594)이 아동 오병합 대역(0.51~0.62) 안에 있어 **안전한 로컬 임계가 존재하지 않음**을
확정했다. 가림·선글라스·옆모습·나이차가 face-pair를 타인 대역으로 끌어내리는 것이 원인이고
([가림·자세 실측](../reviews/2026-07-23-occlusion-pose-split-survey.md)), 모델 교체는 라이선스로
봉쇄돼 있다.

현행 Rekognition 재판정(ADR 030)은 **미배정 얼굴 1장 → 상위 3개 앨범** 구도라 이 문제에 닿지 않는다.
앨범 A와 B가 둘 다 정상 형성되면 훅은 실행조차 되지 않는다. `fragment_hints`는 미배정 얼굴이 다리
역할을 해야 발동하고 로그 전용이다.

## 결정

재군집이 끝난 뒤, **로컬 게이트가 거절한 회색지대 앨범 쌍**을 Rekognition CompareFaces로 재판정해
동일 인물이면 병합한다. ADR 030과 같은 훅 지점·같은 점수 캐시·같은 best-effort 격리를 쓴다.

### ① 후보 선정 — 회색지대만, 확정적 반증은 호출 전에 탈락

앨범 쌍 (A, B)가 다음을 모두 만족할 때만 호출한다:

1. `centroid(A,B) ≥ probe_floor`(0.35) — 정확도가 아니라 **비용 통제**용 바닥이다. 판정은
   Rekognition이 한다. 실측 회수분의 최저 centroid가 0.438이라 0.35에 여유가 있다.
2. 로컬 병합 게이트 미통과 — 통과했다면 이미 병합됐거나 cannot-link로 막힌 쌍이다.
3. **같은 사진 공존 없음** — 두 앨범에 같은 `photo_id`의 (이중 검출 파편이 아닌, 유사도 < 0.95)
   얼굴이 각각 있으면 확정적으로 다른 사람이다(ADR 011). **점수를 묻지 않고 탈락**시킨다: 공짜
   정밀도이자 비용 절감이다. 실측에서는 이 쌍들을 일부러 음성 대조군으로 채점했고, 이것이 판정
   규칙 검증의 그라운드 트루스가 됐다.
4. **사용자 cannot-link 없음** — 사용자가 split으로 갈라놓은 앨범은 점수와 무관하게 탈락한다
   (사람 결정이 기계 증거에 밀리지 않는다).

### ② 판정 규칙 — 대표 K장씩 K×K **전원 합의** (argmax 기각)

앨범당 대표 `reps`(=2, 폭 최대 얼굴 — ADR 030과 같은 규칙·같은 crop 파라미터)를 뽑아 K×K 전부
호출하고, **모든 점수가 `merge_similarity`(90) 이상이고 산포(max−min)가 `max_spread`(5) 이하**일
때만 병합한다.

- **argmax는 실데이터로 기각됐다**: event 128의 확실한 타인 쌍(같은 사진에 함께 찍힌 두 아이) 네
  점수가 `[99.99, 33.74, 1.96, 1.56]`이다. 대표를 1장씩만 뽑았다면 서로 다른 아이의 앨범 두 개가
  통째로 융합됐다. 전원 합의로 바꿔서 잃는 회수량은 1쌍뿐이고, 그 1쌍은 기각 리뷰가 오병합으로
  확정한 event 105의 쌍이라 잃은 것이 아니라 막은 것이다.
- **산포 게이트**는 비용 0의 2차 안전판이다 — 동일인은 어느 대표를 뽑아도 같은 점수가 나오지만
  (실측 통과 13쌍 산포 0.0~1.9, 중앙 0.2) 타인 오답은 한두 쌍만 튀어오른다(대조군 15.0~98.4).
- **임계 90은 95와 결과가 같다**(통과분 전부 98점 이상). 실제 방어선은 임계가 아니라 전원 합의다.
- 얼굴 단위 자동 편입 임계(ADR 030, 90)와 같은 값을 쓰되 **요구 조건이 더 엄격**하다: 앨범 오병합은
  사용자가 일일이 split해야 해서 복구 비용이 얼굴 하나 오배정보다 훨씬 크다.

### ②-b 3개 이상이 합쳐질 때 — 컴포넌트 완전 연결 (체인 융합 차단)

판정은 쌍 단위지만 결과는 전이적이다: A–B가 통과하고 B–C도 통과하면 A·B·C가 한 앨범이 되는데,
**A–C는 아무도 검사하지 않았다.** 이 레포에서 가장 많이 데인 실패 모드가 정확히 이 체인 융합이다 —
event 105에서 서로 다른 아이 5그룹이 33얼굴 하나로 융합된 것도, 파편 병합에 컴포넌트 재평가를 넣은
것도(ADR 024) 전부 같은 원인이다.

따라서 **합쳐질 컴포넌트 안의 모든 쌍이 ②를 통과해야 병합한다**(complete linkage). 유사도 내림차순
greedy로 쌍을 처리하되, 두 컴포넌트를 합치기 전에 교차하는 모든 앨범 쌍의 판정 결과를 확인하고
하나라도 미달·미측정이면 그 병합을 건너뛴다. 미측정 쌍은 후보 조건(①)을 만족하면 호출해서 채우고,
`probe_floor` 미만이라 애초에 후보가 아니었으면 **미달로 취급**한다 — centroid가 0.35에도 못 미치는
두 앨범이 다리를 타고 한 앨범이 되는 것이 바로 막으려는 상황이다.

- 파편 병합이 같은 이유로 같은 선택을 했다(ADR 024는 컴포넌트 전체 재평가, 그 이전 동작은 완전 연결).
  여기서는 판정이 이산값(통과/미달)이라 재평가할 centroid가 없으므로 완전 연결이 자연스러운 형태다.
- 추가 비용은 쌍당 4호출($0.004)로 무시할 수준이다.
- **실측에서는 체인이 관측되지 않았다** — 통과 13쌍 중 같은 앨범을 공유하는 쌍이 없었다(event 117에
  두 쌍이 있으나 네 앨범이 전부 다르다). 즉 이 조항은 관측된 오류의 교정이 아니라, 표본에 없던
  구성에 대한 선제 방어다. 구현 시 합성 픽스처로 고정한다(§검증 계획).

### ③ 병합 반영 — `soft_merge` (must-link 기각)

대표끼리 must-link를 거는 방식은 **금지**한다. must-link는 재군집 초반(② 제약 강제)에 적용돼 대상
클러스터의 응집을 떨어뜨리고, 그 결과 파편병합 게이트 탈락·기존 멤버 축출을 유발한다 — 얼굴 단위
편입에서 이미 겪은 회귀(실 event 134, ADR 030 개정 = `soft_attach` 도입)와 정확히 같은 함정이며,
앨범 단위는 멤버가 많아 피해가 더 크다.

대신 `Constraints.soft_merge`(방향 없는 (rep_a, rep_b) 쌍)를 추가하고, **모든 응집 게이트가 끝난
뒤**(`_recluster_core`의 2차 파편 병합 직후, `_apply_soft_attach` **직전**) 두 대표가 최종적으로
속한 라벨을 union한다:

- 게이트에 불참하므로 대상 클러스터를 재파편화·축출로 흔들지 않는다 — `soft_attach`와 같은 원리.
- **`soft_attach`보다 먼저** 적용한다: 앨범 합치기가 끝난 뒤라야 어태치 대상 클러스터가 최종
  상태가 된다(`_apply_soft_attach`는 `labels[R]`을 읽는다).
- 다음이면 병합을 보류한다(폴백 = 종전대로 갈라진 상태 유지): 두 대표 중 하나가 이번 재군집에서
  노이즈 · 두 클러스터 사이에 `blocking_cannot`(사용자 cannot-link + 같은 사진 자동 cannot-link)이
  하나라도 걸림. 후자는 `_merge_fragments`의 `_sets_blocked`를 그대로 재사용한다.
- ID 승계는 기존 Jaccard 매칭(`_match_cluster_ids`)이 처리한다 — 합쳐진 쪽 중 겹침이 큰 id가
  살아남고 다른 하나는 은퇴한다. 별도 규칙을 두지 않는다.
- 완전 연결 검사(②-b)는 **판정 단계에서 끝낸다** — `soft_merge`로 전달되는 쌍은 이미 컴포넌트
  검증을 통과한 것이므로, `_apply_soft_merge`는 받은 쌍을 union하기만 한다(보류 조건은 위 두 줄뿐).
  판정과 반영을 섞지 않는 것은 `soft_attach`와 같은 구도다.

### ④ 저장 — `.npz` v6 컬럼 `soft_merge_pairs`

`soft_attach_pairs`와 같은 채널로 영속화한다(v5 → v6, 구버전은 빈 튜플 폴백).

- **재전달 안전성**(ADR 007·030): 저장된 `.npz`를 결정적 재군집한 결과와 발행 결과가 항상 일치해야
  한다. 점수 캐시는 "소모품"(손상 시 빈 캐시 취급)이라 캐시만 믿으면 캐시 유실 시 앨범이 도로
  쪼개지는 사용자 가시적 churn이 생긴다.
- 사용자 보정이 우선이다: `handlers`의 `superseded_faces` 프루닝에 `soft_merge_pairs`도 포함해,
  나중 split/reassign이 대표 얼굴을 덮으면 해당 병합 기록을 버린다(`soft_attach_pairs`와 동일 처리).

### ⑤ 비용 — 기존 점수 캐시 재사용 + job당 상한

- 점수 캐시는 ADR 030의 `rekognition-scores/{event_id}.json`을 그대로 쓴다(`(face_id, face_id) →
  Similarity`). **대표가 바뀌지 않는 한 재군집이 몇 번 돌아도 0원**이다.
- 실측 기반 추정(아이 20명 이벤트, 30일): 최초 1회 $0.19, 이후 월 **$1.48**. 대표 교체 기대 횟수가
  `K·ln(최종/초기)`라 **사진 장수와 거의 무관**하다(하루 30장이든 300장이든 같다).
- 앨범 쌍 수는 아이 수의 제곱으로 는다(5명 $0.01 → 50명 $1.22). ADR 030과 별도로
  `REJUDGE_PAIR_MAX_CALLS_PER_JOB` 상한을 둔다. 초과분은 판정 보류 + `truncated` 로그(침묵 절단 금지).
- 얼굴 미검출(`InvalidParameterException`)은 ADR 030과 같이 `-1.0` 센티널로 캐싱해 영구 재과금을
  막는다. 일시 장애(스로틀)는 캐싱하지 않는다.

### ⑥ 격리

ADR 030과 동일한 best-effort: 재판정·캐시의 어떤 실패도 job을 죽이지 않으며, 실패 시 1차 패스 결과
그대로(현 동작 유지)다. 병합 `soft_merge`를 반영한 2차 패스가 성공한 뒤에만 event 교체를 커밋한다 —
예상 밖 모순 제약이 저장되면 이후 모든 재군집이 죽는 오염을 막는다.

## 설정 (`Settings` → `RejudgeConfig` 확장)

| 키 | 기본값 | 뜻 |
|----|--------|-----|
| `REJUDGE_PAIR_ENABLED` | `false` (초기 롤아웃) | 앨범 쌍 재판정 활성 — 킬 스위치 |
| `REJUDGE_PAIR_PROBE_FLOOR` | `0.35` | 이 centroid 미만 앨범 쌍은 묻지 않음 (비용 통제) |
| `REJUDGE_PAIR_MERGE_SIMILARITY` | `90.0` | K×K **전원**이 이 값 이상이어야 병합 |
| `REJUDGE_PAIR_MAX_SPREAD` | `5.0` | K×K 산포(max−min) 상한 — 2차 안전판 |
| `REJUDGE_PAIR_REPS` | `2` | 앨범당 대표 수 K (쌍당 K² 호출) |
| `REJUDGE_PAIR_MAX_CALLS_PER_JOB` | `200` | job당 호출 상한 (아이 수 제곱 증가 안전판) |

## 롤아웃

ADR 030이 밟은 순서를 그대로 따른다: **로그 전용 → 제안(Spring 계약) → 자동 병합**. 초기 배포는
`REJUDGE_PAIR_ENABLED=false`이며, 실사용자 이벤트에서 후보 로그를 며칠 관측해 회색지대 비율·오탐이
실측(아동 치우침 코퍼스)과 어긋나지 않는지 확인한 뒤 켠다.

## 기각한 대안

- **로컬 face-pair 바닥 완화** — 실서버 전수 육안 채점으로 기각 확정
  ([기각 리뷰](../reviews/2026-07-23-merge-facepair-relaxation-rejected.md)). 겹치는 대역에는 임계가 없다.
- **대표 1쌍 argmax 판정** — event 128 대조군 `[99.99, 33.74, 1.96, 1.56]`으로 실측 기각(위 ②).
- **must-link로 병합 반영** — event 134 회귀와 같은 함정(위 ③).
- **`fragment_hints`를 자동 병합으로 승격** — 거의 공짜지만 미배정 얼굴이 다리로 있어야 발동해
  커버리지가 좁다. 이 결정의 후보 소스 중 하나로 흡수한다(별도 채널로 두지 않음).
- **Rekognition Collection(IndexFaces/SearchFaces)으로 군집 대체** — 쪼개짐을 원리적으로 해결하고
  조회 단가도 싸지만, 얼굴 특징벡터를 AWS에 **영속 저장**하게 되어 생체정보 위탁·삭제 의무의 무게가
  완전히 달라진다. CompareFaces가 호출 후 아무것도 남기지 않는다는 점이 현 개인정보 논리의 기반이라
  별개 결정 사안이다. 전체 재군집 결정성(ADR 007) 아키텍처도 전면 재작성이 필요하다.

## 검증 계획 (구현 시)

- `python -m app.pipeline.rejudge` — 앨범 쌍 판정 로직: 전원 합의 통과/1개 미달 기각, 산포 초과 기각,
  같은 사진 공존 호출 전 탈락(호출 0회), 사용자 cannot-link 탈락, 상한 절단 + `truncated`,
  캐시 히트 0호출. **체인 차단**: A–B·B–C가 통과해도 A–C가 미달이면 셋이 합쳐지지 않고
  (A–B만 병합), A–C가 `probe_floor` 미만이라 미측정이면 미달로 취급해 역시 합쳐지지 않는다.
- `python -m app.pipeline.cluster` — `_apply_soft_merge`: 게이트 이후 union이 다른 앨범의 멤버십을
  바꾸지 않음(응집 불변), 대표가 노이즈면 보류, `blocking_cannot` 걸린 쌍 보류, `soft_attach`보다
  먼저 적용됨, 고정점 수렴.
- `python -m app.handlers` — 종단: 병합 → npz `soft_merge_pairs` 저장 → 재분류 시 추가 호출 0회로
  같은 병합 유지, 사용자 split이 병합을 되돌림, 전면 장애 = 비활성 동일 결과.
- `python -m app.worker --smoke` — 발행 경유 왕복.
- **실 이벤트 회귀**: 실측이 남긴 회수 12개 이벤트(104·105·108·111·114·117·121·123·127·134·135·51)가
  붙고, 적대 이벤트(105 체인·121·131 남매·128)가 붙지 않는지 파티션 diff로 확인.

## 한계

- 실측 표본이 아동·테스트 이벤트에 치우쳐 있다 — 회색지대 비율 30%·미배정 39%는 실사용자
  이벤트에서 재측정해야 한다(과금 추정의 (a)는 미배정 비율에 선형).
- 원본이 소실된 이벤트는 crop을 만들 수 없어 판정 불가다(ADR 030과 같은 한계, S3 버저닝 필요).
- 대조군은 같은 이벤트 안의 타인만 담는다 — 이벤트 격리(ADR 007) 덕에 실사용 경로엔 없는 조합이지만,
  "닮은 남남이 각자 앨범을 가진" 최악 케이스의 표본은 남매 3쌍(event 131)뿐이다.
- **완전 연결(②-b)은 보수적인 쪽으로 틀린다**: 한 인물이 3개로 쪼개졌는데 그중 한 쌍만
  `probe_floor` 미만(예: 정면 앨범 ↔ 완전 측면 앨범)이면 나머지 두 쌍이 100점이어도 셋 다 합쳐지지
  않는다. 덜 합쳐진 앨범(reassign 한 번)이 잘못 합쳐진 앨범(일일이 split)보다 복구가 싸다는 기존
  판단(기각 리뷰 §기각 근거)을 따른 것이며, 실측 표본에는 이 구성이 없어 손실 규모는 미측정이다.
- 쌍둥이·형제가 각자 앨범을 가진 이벤트는 여전히 잔여 위험이다. 관측 시 `merge_similarity`를 95로
  올리거나 해당 환경만 비활성으로 운용한다.
- K=3의 추가 이득·비용은 미측정(산포 게이트가 K=2에서 이미 갈라서 우선순위 낮음).
