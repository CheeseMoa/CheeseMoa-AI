# 파편병합 face-pair 완화 실서버 재검증 — 0.475 유지 확정, '최강쌍 OR'·'평균 하향' 기각 (2026-07-23)

- 날짜: 2026-07-23
- 도구: 실이벤트 스윕(스크래치, 하이브리드 게이트는 임시 구현 후 되돌림) ·
  `scripts/judge_event_groups.py`(이벤트 그룹별 얼굴 crop 몽타주 — 병합 육안 채점, 재사용 가능) ·
  `scripts/eval_accuracy.py`(라벨 코퍼스 회귀 하네스)
- 데이터: S3 `cheesemoa-dev/embeddings/` 실이벤트 48개 전수 + 원본 이미지 crop 육안 판독
- 관련: [ADR-016](../decisions/016-merge-facepair-cohesion-gate.md)(이 임계의 제정·재보정 이력) ·
  [2026-07-23 가림·자세 갈라짐 실측](2026-07-23-occlusion-pose-split-survey.md)(같은 날, 갈라짐 원인 조사)

**판정: `merge_facepair_floor` 0.475를 유지한다. 가림·스타일 파편을 회수하려는 두 완화안 —
① 평균 미달이어도 교차 최강쌍이 높으면 병합(OR 게이트), ② 평균 바닥 0.45 하향 — 은 모두 실서버
데이터 육안 채점으로 기각. 정당 병합의 최강쌍(0.524)이 서로 다른 아이 간 최강쌍 대역(0.51~0.62)
안에 있어 안전한 임계가 존재하지 않는다. event 133류 파편의 정답 경로는 병합 완화가 아니라
uncertain → 사용자 reassign(must-link)이다.**

## 문제 — event 133 (group 44): 카리나 모자·안경 2장이 "분류가 어려워요"로

실서버 리포트를 층별로 추적한 결과(수치는 재현 검증 완료):

1. 모자 컷·안경 컷은 서로 0.547로 닮아 HDBSCAN이 2장짜리 미니 클러스터를 만들었다.
2. 카리나 본진(5얼굴)과의 파편 병합 판정: centroid 0.625로 임계(0.55) 통과, **face-pair 평균
   0.447 < 0.475로 0.028 차 차단** (가림이 개별 유사도를 타인 대역으로 끌어내림 — 같은 날
   가림·자세 실측 리뷰의 조건별 붕괴 폭과 일치).
3. 저신뢰 축출에서 두 얼굴 다 자기 미니 클러스터(LOO 0.547)와 카리나 본진(0.543/0.556) 사이
   마진 ±0.01 → "두 인물 사이 애매"(ambiguous) 판정 → 미니 앨범 해체 → uncertain.

즉 본진과 **너무 가까웠기 때문에** 앨범이 해체된 케이스다. 붙일 방법이 없나? → 병합 게이트 완화
2안을 실서버 데이터 전수로 검증했다. (노이즈 쪽은 같은 날 배포된 margin 구제 게이트가 이미 이
이벤트에서 초소형 얼굴·행사 컷 2장을 정확히 회수해 피해를 5장→3장으로 줄였다. 남은 핑크머리 1장은
top1 카리나 0.416 vs top2 고윤정 0.314, 배율 1.32<1.7로 margin 게이트가 의도대로 보류.)

## 방법 — 후보 임시 구현 + 실이벤트 48개 파티션 diff + crop 육안 채점

`_merge_fragments`에 '평균 OR 최강쌍' 게이트를 임시 구현하고, 서버 현행 설정(margin ON)을
기준(BASE)으로 후보 4종을 48개 이벤트 전수에 돌려 **파티션이 달라지는 이벤트만** 추출했다.
달라진 병합은 전부 원본에서 얼굴을 crop해 그룹별 몽타주로 육안 판정했다(`judge_event_groups.py`).

| 후보 | 내용 | 파티션 변화 |
|------|------|------------|
| HYB47/50/53 | 평균 0.475 유지 + 최강쌍 ≥0.47/0.50/0.53이면 OR 통과 | 23 / 23 / 22개 이벤트 |
| MEAN45 | 평균 바닥 0.475 → 0.45 (ADR-016 재보정 기각안) | 4개 이벤트 |

## 결과 — 육안 채점 (HYB50 기준)

**정당한 병합 (3건)**: event 133(카리나 모자·안경 2장 → 본진 합류, 목표 케이스) ·
event 51(동일 여성 스타일 파편 8+2, 평균 0.455/최강 0.594) · event 117(동일 유아 추정 2+3).

**오병합 (3건+, 전부 육안 확정)**:

- **event 105**: 남아 2계열·여아 2계열·아기 — **명백히 서로 다른 아이 5그룹이 33얼굴 하나로
  체인 융합**. 그룹 간 최강쌍 0.53~0.58이 다리가 됐다.
- **event 121**: 아기 + 여아 파편 + 수박모자 유치원생 등 **다른 아이 다수가 25얼굴로 융합**.
- **event 131**(왕실 아동 테스트셋): **오스카르·에스텔 남매 융합**.
- event 127(여배우 2인 의심, 최강 0.623)은 판정 보류지만 위험 신호.

이벤트 84~99 계열은 원본이 S3에서 삭제돼 육안 판정 불가(부수 발견 참조). MEAN45는 event 105
체인을 다시 열어 ADR-016 재보정의 기각 사유가 실서버에서 그대로 재확인됐다.

## 기각 근거 — 겹치는 대역에는 임계가 없다

정당 병합의 최강쌍: 0.524(133) · 0.594(51). 오병합의 최강쌍: **0.51~0.62**(105 체인·121·131 남매).
아이 얼굴은 서로 다른 아이끼리도 우연한 강한 쌍이 0.5를 넘는 일이 흔해서, event 133을 구하는
어떤 최강쌍 임계도 아동 체인 융합을 함께 연다. ADR-016이 min/max가 아닌 **평균**을 게이트로 택한
설계가 실서버 데이터로 재입증된 것. 잘못 합쳐진 앨범(사용자가 일일이 split)이 덜 합쳐진 앨범
(reassign 한 번)보다 복구 비용이 훨씬 크므로 보수 쪽이 맞다.

구현은 전량 되돌렸다(코드 무변화). 향후 같은 완화 아이디어가 나오면 이 문서의 이벤트
(105·121·131)로 먼저 검증할 것.

## 부수 발견 — 검증용 원본 소실

이벤트 84~99 계열은 `.npz`(임베딩·bbox·s3_key)는 남아 있으나 **원본 이미지가 S3에서 삭제돼**
crop 육안 검증이 불가능했다. 임계 재보정·오병합 조사는 원본 육안 판독이 최종 근거이므로, 검증
표본이 계속 줄어든다. S3 버저닝/보존 정책(2026-07-11 후속의 P0 잔여 항목과 동일 주제)을 Spring과
합의해 둘 것.
