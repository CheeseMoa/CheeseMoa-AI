# 크로스-이벤트 신원 연결 임계 측정 — 자동 0.72 / 제안 0.55 확정 (2026-07-22)

같은 사람이 새 이벤트에 등장하면 "인물 N"으로 새로 생기는 문제(백엔드 논의)의 해법으로 "이벤트
격리 유지 + 그룹 인물 명부 연결층"을 검토하면서, 그 연결층의 임계를 라벨 코퍼스와 **실서버 38개
이벤트 전수**로 측정했다. **판정: 자동 연결은 centroid 유사도 0.72 이상(실측 타인 천장 0.687 +
마진 0.033), 0.55~0.72는 "같은 분인가요?" 제안 + 사용자 확인, 0.55 미만은 새 인물. 완전 자동은
어떤 임계로도 불가능하다 — 타인(닮은꼴 유아·모녀)과 동일인(연령차)이 0.55~0.69에서 원리적으로
겹친다.**

전제: 격리 단위를 이벤트 → 그룹으로 바꾸는 전면 개편은 반대로 결론났다(결함 격리 상실, n² 거리행렬
메모리, 이벤트 전제 제약 기계 전면 수정). 연결층은 이벤트 재군집 결과의 클러스터 centroid를 그룹
명부와 대조하는 얇은 추가 단계다.

## 방법 — 3단 측정

**① 라벨 코퍼스 크로스-세션 (`scripts/survey_cross_event_distance.py`)**: `image/images/child/`
파일명 `{ID}_{연}_{월}_…` 구조가 신원 8명 × 세션(연_월) 라벨을 제공(교차연령 아동 = 최악 하한) +
`similar/similar2/similar3` 단일 신원. 같은사람 크로스-세션 149쌍, 타인 1,982쌍.

**② 임베더 A/B (`scripts/survey_cross_event_adaface.py`)**: 같은 얼굴·같은 쌍에 AuraFace(운영)와
AdaFace IR-101/WebFace12M(비상용 yardstick, HF 캐시본)를 동시에 얹어 임베더 효과만 격리.

**③ 실서버 확정 측정 (`scripts/xevent_pair_sheets.py` + vision 판독)**: S3 38개 이벤트 .npz의
클러스터 115개 → 재업로드 확실 링크(크로스-이벤트 멤버쌍 유사도 ≥0.985 = 같은 사진)로 **신원
그룹 35개** 자동 결속 → 그룹쌍 중 centroid ≥0.45인 49쌍 전수를 얼굴 crop 비교 시트로 육안
판독(ADR-017 vision 워크플로우) + 결정적 2쌍은 데이터 소유자 확인. 같은사진 동시 등장(ADR-011
타인 확정) 교차검증 병행. 확정 라벨로 크로스-이벤트 클러스터 쌍 6,253개를 같은사람/타인으로 분류.

## 결과 ① — 라벨 코퍼스: 완전 분리 불가, 모델 업그레이드는 recall 2배·꼬리 불변

| | 같은사람 min/med/max | 타인 med/**max** | 0.60 임계 recall/precision |
|---|---|---|---|
| AuraFace | 0.175 / 0.560 / 0.812 | 0.251 / **0.604** | 33% / 98% |
| AdaFace | 0.348 / 0.636 / 0.854 | 0.099 / **0.602** | **58%** / 99% |

AdaFace는 타인 중앙값을 절반 이하로 눌러 같은 안전선에서 **recall이 약 2배**(0.60 기준 33→58%,
0.55 기준 54→68%)이지만, **타인 최댓값(꼬리)은 0.604→0.602로 불변** — 판별력 향상은 쉬운 중간을
가르지, 닮은꼴 하드케이스를 못 없앤다. 유료 라이선스 검토(CLAUDE.md 선행조건 ⓐ의 자체 데이터
A/B)에 그대로 쓸 수 있는 수치다: ROI = "제안·자동연결량 2배", 한계 = "완전 자동은 유료로도 불가".

## 결과 ② — 실서버: 타인 천장 0.687, 동일인 새-세션 0.85~0.93, 사이가 빈 구간

35개 신원 그룹의 육안 판독(49쌍) 확정 라벨 기준:

| 분포 | n | med | p99 | **max** |
|---|---|---|---|---|
| 같은사람 (재업로드 포함) | 546 | 0.984 | — | 1.000 |
| 같은사람 진짜 새-세션 (<0.95) | 232 | — | — | **0.934** |
| 타인 | 5,707 | 0.234 | 0.610 | **0.687** |

- **타인 천장 0.687** = 닮은꼴 유아 쌍(더벅머리 남아 vs 단발 여아 — 명백한 타인). 차상위 0.667,
  이후 0.65대 밀집. 성인 무관계 쌍은 대부분 0.5 미만.
- **모녀 0.59**: 옛날 인화 여성(어머니 젊을 적) vs 현재 여성 딸 — 가족 서비스의 세대 닮은꼴이
  실제로 이 높이까지 온다. 절대 자동 병합하면 안 되는 대역.
- **동일인 연령차 0.55~0.66**: 같은 아이의 아기 때 vs 최근(소유자 확인 완료 0.656 포함) — 타인
  대역과 완전히 겹친다. **이 구간은 임계로 못 가르며, 사용자 확인이 유일한 판별기다.**
- **동일인 진짜 새-세션(같은 사람, 다른 사진 배치) 0.848~0.934**: 소유자 확인 완료(0.848 셀피
  배치 쌍). 실사용 패턴(비슷한 시기 사진)은 라벨 코퍼스보다 훨씬 높게 뭉친다.
- 0.69~0.85가 빈 구간이다 — 자동 임계를 이 안에 두면 실측 오탐 0.

임계 스윕 (실서버, 클러스터 쌍):

| 임계 | 타인 오탐 | 같은사람 자동연결 | 진짜 새-세션 연결 |
|---|---|---|---|
| 0.55 | 170쌍 | 92% | 80% |
| 0.60 | 70쌍 | 85% | 64% |
| 0.65 | 2쌍 | 71% | 32% |
| 0.70 | 0쌍 | 68% | 25% |
| **0.72** | **0쌍** | **68%** | **24%** |
| 0.75 | 0쌍 | 64% | 15% |

0.72 채택 근거: 0.70은 천장과 마진 0.013으로 얇고, 0.75는 마진 +0.03을 위해 새-세션 연결을
24→15%로 깎는다. 0.72는 마진 0.033에 recall 손실이 0.70 대비 1%p.

## 확정 계약 (연결층 구현 시)

- **≥ 0.72: 자동 연결** — 기존 그룹-인물 id 승계. 실측 오탐 0 구간.
- **0.55 ~ 0.72: 제안** — "이 사람, ○○님과 같은 분인가요?" UI로 사용자 확인 후 must-link 반영.
  모녀(0.59)·닮은꼴 유아(0.55~0.69)·동일인 연령차(0.55~0.66)가 전부 이 구간에 떨어진다.
- **< 0.55: 새 인물** — 제안도 하지 않는다(라벨 코퍼스 기준 제안 정확도가 0.55 아래에서 급락,
  0.50에서 74%).

## 한계

- 신원 그룹의 뼈대가 재업로드 링크(≥0.985)라 코퍼스 특성상 같은사람 표본이 재업로드에 치우침 —
  진짜 새-세션 같은사람은 성인 1명(4개 배치)뿐이다. 새-세션 하한 0.848은 표본 1명의 값이며,
  조명·연령 변화가 큰 실사용에서는 더 낮게 나올 수 있다(그 경우 제안 구간이 받는다 — 안전 방향).
- 제안 구간의 실서버 정밀도(≈10~18%)는 이 코퍼스의 낮은 base rate(대부분 타인 쌍) 산물이다.
  실사용은 "같은 그룹 = 같은 모임 반복"이라 base rate가 높아 정밀도도 오른다(라벨 코퍼스 86%가
  상한 참고치).
- 유아 그룹 2쌍(저해상 아기 vs 확정 동일아, 아기 vs 모자 소년)은 미확인으로 타인 처리 — 타인
  분포를 두껍게 잡는 보수 방향이라 임계에 무해.
- 타인 천장 0.687의 마진 0.033은 닮은꼴 형제자매 미표집을 감안하면 얇다. 배포 후 자동 연결
  오류 리포트 1건이라도 오면 0.75로 상향(-9%p recall)이 1차 대응이다.

## 후속

1. 연결층 설계·티켓화: 그룹 인물 명부 저장(위치·스키마), classify 결과의 `is_new`/그룹 인물 id
   의미 확장, Spring 계약 합의(제안 UI 플로우 포함).
2. 제안 확인 UI가 없는 동안 자동 연결(≥0.72)만 먼저 켜는 단계적 출시 가능 — 오탐 0 실측 구간만
   쓰므로 안전.
3. 유료 임베딩 라이선스 검토 시 이 문서의 A/B 수치를 선행조건 ⓐ의 근거로 사용.
