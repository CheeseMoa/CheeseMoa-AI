# 원본 해상도 눈 crop 실측 — 기각 (2026-07-17)

## 가설

눈감음 판정의 미탐(event 61: 양눈 감은 545px 얼굴의 한쪽 눈 closed 0.133)이 입력 경로의 정보
손실 때문일 것이다 — 현행은 얼굴을 112×112로 정렬 축소한 뒤 24px 눈 박스를 32×32로 재확대해
CNN에 넣으므로(이중 리샘플링 + 24px 병목), 큰 얼굴일수록 눈 디테일이 뭉개진다. Umeyama 변환과
눈 창 변환을 아핀 합성해 **원본에서 32×32 눈 crop을 한 번의 warp로 직접** 뽑으면(리샘플링 1회,
유효 해상도 24→32px, align.py와 동일한 배율 프리블러) 판정이 개선될 것이다.
"입력 품질이 모델보다 먼저다" 원칙에 따라 모델 교체 전 실측했다. 도구:
`scripts/survey_native_eyecrop.py`.

## 실측 (871 얼굴 — 실 이벤트 + 로컬 코퍼스 + event 61, 프로덕션 판정 의미론)

- 눈별 |Δclosed| 분포: 중앙값 0.000 / p90 0.165 / 최대 0.989. **얼굴폭 >300px 구간이 가장
  변화가 작다**(p90 0.039 — <112px 구간 0.200) — "축소가 큰 얼굴을 해친다"는 가설의 예측과 반대.
- 판정 뒤집힘 2건, 둘 다 악화 (육안 확인):

| 사례 | 실제 | 기존 | 원본 직접 | 결과 |
|---|---|---|---|---|
| event 61 미탐 (보정 스톡, 감음) | 감음 | [1.000, 0.133] 미탐 | [1.000, **0.004**] | 목표 사례가 **더 나빠짐** |
| 안경 남성 (감음, 기존 정탐) | 감음 | [0.999, 0.990] ✅ | [1.000, **0.001**] | **정탐 붕괴** |
| 웃는 소녀 (뜸) | 뜸 | [0.472, 0.998] 미판정 ✅ | [**0.999, 1.000**] | **새 오탐** — 웃음 실눈 오탐을 원본 crop이 유발 |

## 판정 — 기각, 원인 해석

미탐의 원인은 입력 해상도가 아니라 **CNN의 도메인**이다. open-closed-eye-0001은 저해상도·소프트한
운전자 모니터링 입력으로 학습됐고, 현행 경로의 이중 리샘플링(112 축소 → 24px 박스 → 32 확대)이
만드는 부드러운 입력이 오히려 학습 분포에 **정합**한다. 원본 직접 crop의 선명한 입력(속눈썹·안경
테·화장 디테일)은 분포 밖이라 판정이 무작위로 흔들린다 — 감은 눈을 뜬 것으로(정탐 붕괴), 웃음
실눈을 감은 것으로(신규 오탐, 그동안 관측되지 않던 웃음 캔디드 오탐이 이 경로에서 등장).

따라서 **현행 입력 경로를 유지한다**. 24px 병목은 손실이 아니라 사실상의 도메인 매칭이었다.
남은 눈감음 정확도 개선 경로는 입력이 아니라 모델/임계 차원이다: ① 실서비스 오탐·미탐 리포트로
라벨셋 축적 → 임계·양눈 규칙 재보정(다음 구현 목표 3), ② 부족하면 상용 가능 대체 신호(예:
MediaPipe FaceMesh 눈꺼풀 랜드마크 EAR, Apache 2.0) 검토.
